## 1. 引言:为什么AI PM必须区分线框图与模型

在AI产品开发流程中,**设计保真度**(Design Fidelity)的选择直接影响团队效率与决策质量。一个常见场景:

**错误示例**:产品经理用高保真模型召开需求评审会,工程师被精美UI吸引,忽略核心逻辑漏洞——会后发现信息架构存在根本缺陷,返工成本激增。

**正确姿势**:先用线框图对齐需求与用户流程,确认"骨架"无误后,再用模型展示视觉方案,分阶段推进设计决策。

本章核心价值:
- **效率提升**:选对工具,AI产品MVP开发周期缩短30%
- **决策质量**:避免"视觉先行"陷阱,聚焦用户价值
- **协作语言**:建立产品、设计、工程三方共同话语体系

对于AI产品经理,线框图是**逻辑验证工具**,模型是**价值传达媒介**——两者配合使用,才能在技术复杂度与商业价值间找到平衡。

## 2. 线框图详解 | Wireframes Explained

### 2.1 定义与特征

**线框图**(Wireframe)是产品界面的**低保真结构图**(Low-Fidelity Structural Blueprint),类似建筑设计中的平面图——只标注房间布局与动线,不涉及装修风格。

**核心特征**:
| 维度 | 描述 |
|------|------|
| **视觉样式** | 黑白灰色系,无品牌元素 |
| **内容呈现** | 占位符代替真实内容(如"Lorem Ipsum"文本块) |
| **交互展示** | 标注按钮位置与页面跳转逻辑 |
| **制作工具** | 纸笔、白板、低保真设计软件 |
| **迭代速度** | 快速,5-30分钟完成单页 |

### 2.2 核心价值

**对AI产品的特殊意义**:
1. **信息架构验证**:大模型对话界面的历史记录展示逻辑
2. **功能优先级排序**:AI推荐系统的调控面板核心参数位置
3. **跨角色沟通**:产品经理与算法工程师对齐"模型输出展示方式"

**案例**:某AI客服系统线框图设计
```
┌─────────────────────────────────┐
│ [Logo] 智能客服助手        [设置] │
├─────────────────────────────────┤
│ 对话历史(左侧折叠菜单)             │
│ ├─ 今日对话                       │
│ ├─ 本周对话                       │
│ └─ 历史归档                       │
│                                  │
│ 主对话区                          │
│ ┌──────────────────────┐        │
│ │ 用户: 如何退款?        │        │
│ │ AI: [生成中...]        │        │
│ └──────────────────────┘        │
│                                  │
│ [输入框] [发送]  [满意度评价]     │
└─────────────────────────────────┘
```

**线框图聚焦的3个问题**:
- **What**:界面需要哪些元素?(对话框、输入框、评价按钮)
- **Where**:元素如何排列?(左侧历史,中部对话,底部输入)
- **How**:用户如何操作?(点击发送→AI生成→评价)

### 2.3 适用场景

✅ **应该使用线框图**:
- 需求评审会议(快速对齐功能范围)
- 早期用户调研(测试信息架构是否合理)
- 敏捷开发迭代(每周Sprint规划)
- 技术可行性讨论(工程师评估实现难度)

❌ **不应使用线框图**:
- 向投资人展示产品愿景(缺乏视觉冲击力)
- 品牌形象定位会议(无法体现设计语言)
- 最终用户测试(用户难以想象真实产品)

### 实战洞察 💡

**AI产品线框图的3个独特挑战**:

1. **动态内容展示**:大模型输出不可预测,需设计"加载态"与"错误态"占位
   - 解决方案:在线框图中明确标注`[AI生成中...]`、`[生成失败,点击重试]`

2. **多模态交互**:文本、图片、语音混合输入,布局复杂度提升
   - 解决方案:用分层线框图,分别展示"纯文本模式"与"多模态模式"

3. **个性化推荐逻辑**:千人千面的推荐内容,需标注"算法决策点"
   - 解决方案:在线框图中用虚线框标注`[个性化推荐区,算法动态填充]`

## 3. 模型详解 | Mockups Explained

### 3.1 定义与特征

**模型**(Mockup)是产品界面的**高保真视觉稿**(High-Fidelity Visual Representation),类似装修后的样板房——展示真实色彩、材质、家具摆放。

**核心特征**:
| 维度 | 描述 |
|------|------|
| **视觉样式** | 完整品牌色系、字体规范、图标库 |
| **内容呈现** | 真实或接近真实的文案、图片 |
| **交互展示** | 静态截图或简单点击跳转(非完整交互) |
| **制作工具** | 专业设计软件(即时设计、MasterGo、Pixso) |
| **迭代速度** | 中等,单页需1-4小时(含视觉调优) |

### 3.2 核心价值

**对AI产品的特殊意义**:
1. **品牌差异化**:在同质化AI工具中建立视觉识别度
2. **信任感塑造**:专业设计提升用户对AI能力的心理预期
3. **多端适配验证**:展示响应式设计在不同设备的表现

**案例**:AI写作助手模型设计要点
```
视觉层次设计:
┌─────────────────────────────────────────┐
│ [渐变Logo] AI写作助手  [头像▼]           │ ← 品牌主色调(#4F46E5紫罗兰)
├─────────────────────────────────────────┤
│ 📝 新建写作  📚 我的文档  🎨 风格库       │ ← Tab Bar采用卡片阴影设计
├─────────────────────────────────────────┤
│ ┌─────────────────────────────────┐    │
│ │ 标题: 产品发布会邀请函           │    │ ← 输入框采用圆角矩形+微光效果
│ │ 风格: 正式商务 ▼                │    │
│ │ 语气: 热情友好 ▼                │    │
│ │                                  │    │
│ │ [AI生成内容区]                   │    │
│ │ 尊敬的合作伙伴:                  │    │ ← 生成文本采用渐显动画
│ │ 我们诚挚邀请您参加...            │    │
│ └─────────────────────────────────┘    │
│                                          │
│ [⚡重新生成] [✨优化] [💾保存] [📤分享]  │ ← CTA按钮采用微渐变+悬浮动效
└─────────────────────────────────────────┘
```

**模型展示的3个层次**:
- **品牌层**:Logo、色系、字体传达产品调性
- **功能层**:图标、按钮、表单展示操作路径
- **情感层**:插画、动效、留白营造使用氛围

### 3.3 适用场景

✅ **应该使用模型**:
- 管理层评审(CEO/CTO需看到"成品感")
- 投资人路演(BP中的产品截图)
- 用户可用性测试(真实视觉环境下的交互测试)
- 开发交付(设计师向工程师交付切图与标注)

❌ **不应使用模型**:
- 早期头脑风暴(制作成本过高,限制创意)
- 快速A/B测试方案(视觉调优耗时,影响迭代速度)
- 纯功能逻辑讨论(视觉细节干扰决策焦点)

### 实战洞察 💡

**AI产品模型设计的5个关键决策**:

1. **置信度可视化**:大模型输出需标注可信度
   - 设计方案:在生成内容右上角显示`可信度:87% ●●●○○`

2. **多轮对话视觉差异**:区分用户输入与AI回复
   - 设计方案:用户气泡采用浅色系,AI气泡采用品牌主色调

3. **复杂参数调节界面**:平衡专业性与易用性
   - 设计方案:默认隐藏高级参数,提供"简洁模式/专家模式"切换

4. **加载状态设计**:AI生成耗时需友好等待体验
   - 设计方案:骨架屏(Skeleton Screen)+百分比进度条

5. **错误提示人性化**:避免技术术语吓跑非技术用户
   - 错误示例:`Error 500: Token limit exceeded`
   - 优化方案:`😅 内容有点长,帮你精简一下吧~`

## 4. 对比分析 | Comparison Analysis

### 4.1 核心差异对比表

| 对比维度 | 线框图 Wireframe | 模型 Mockup |
|---------|-----------------|------------|
| **保真度** | 低保真(Low-Fi) | 高保真(High-Fi) |
| **视觉元素** | 黑白灰+占位符 | 完整色彩+真实内容 |
| **设计重点** | 结构、布局、逻辑流程 | 品牌、美学、视觉层次 |
| **制作时间** | 5-30分钟/页 | 1-4小时/页 |
| **修改成本** | 极低(快速擦除重画) | 中等(需调整图层与样式) |
| **主要受众** | 产品团队、工程师 | 管理层、用户、投资人 |
| **测试目标** | 功能完整性、操作流程 | 视觉吸引力、品牌认知 |
| **类比** | 建筑平面图 | 装修样板间 |
| **AI产品特殊价值** | 验证"算法输出逻辑" | 展示"AI能力可信度" |

### 4.2 决策流程图

```
产品需求诞生
    ↓
┌───────────────────────────────┐
│ 问题1: 功能逻辑是否明确?       │
└───────────────────────────────┘
    │ 否 → 【使用线框图】
    │       ├─ 画出用户操作流程
    │       ├─ 团队讨论优化方案
    │       └─ 迭代至逻辑清晰
    │
    ↓ 是
┌───────────────────────────────┐
│ 问题2: 是否需要stakeholder认同? │
└───────────────────────────────┘
    │ 是 → 【制作模型】
    │       ├─ 应用品牌设计规范
    │       ├─ 准备演示Demo
    │       └─ 收集反馈调整
    │
    ↓ 否
┌───────────────────────────────┐
│ 问题3: 开发资源是否紧张?       │
└───────────────────────────────┘
    │ 是 → 【线框图+原型工具】
    │       └─ 跳过模型直接开发
    │
    ↓ 否
    【完整设计流程】
    线框图 → 模型 → 交互原型 → 开发
```

### 4.3 使用时机矩阵

| 场景 | 早期探索 | 需求评审 | 用户测试 | 投资人展示 | 开发交付 |
|------|---------|---------|---------|-----------|---------|
| **线框图** | ✅ 推荐 | ✅ 推荐 | ⚠️ 可选 | ❌ 不推荐 | ⚠️ 配合使用 |
| **模型** | ❌ 成本高 | ⚠️ 可选 | ✅ 推荐 | ✅ 必需 | ✅ 必需 |

**图例说明**:
- ✅ 推荐:该阶段首选工具
- ⚠️ 可选:根据项目资源决定
- ❌ 不推荐:性价比低或效果不佳

### 5.2 工具选型决策树

**根据团队规模选择**:
```
团队规模

│
├─ 6-30人(成长期)

└─ 30人+(成熟企业)
   └─ 推荐: MasterGo + 蓝湖(交付管理)
      理由: 500人协同支持,设计系统管理完善

**根据项目类型选择**:
```
项目类型
├─ AI对话产品(如ChatBot)
│  └─ 推荐: 墨刀(快速验证对话流程) → Pixso(多模态界面设计)
│
├─ AI数据分析平台
│  └─ 推荐: MasterGo(复杂Dashboard设计) + 即时设计(组件库管理)
│
└─ AI内容生成工具
   └─ 推荐: Pixso(AI生成UI加速迭代) + 墨刀(交互原型测试)
```

### 5.3 工具详解

#### 5.3.1 即时设计 (Instant Design)

**官方定位**:完美适应各种系统的一站式在线UI设计平台(据即时设计官网)

**核心优势**:
- **资源广场**:19K+优秀作品、200+资源库免费调用
- **文件兼容**:支持Figma、Sketch、XD文件在线转换
- **开发模式**:自动生成标注,完整设计交付流程
- **多平台支持**:网页、macOS、Windows、Linux、iOS、Android、微信小程序

**适合场景**:
- ✅ 需要快速借鉴优秀设计案例
- ✅ 团队使用多种设计工具需要文件互转
- ✅ 重视设计到开发的无缝对接

**AI产品使用建议**:
- 在资源广场搜索"AI对话"、"数据看板"等关键词,复用成熟模板
- 利用开发模式的自动标注功能,减少与工程师的沟通成本

#### 5.3.2 MasterGo (莫高设计)

**官方定位**:AI时代的数字界面生产平台,字节跳动旗下火山引擎发布(据MasterGo官网)

**核心优势**:
- **原型能力**:从低保真到高保真原型的全流程支持
- **交互设计**:简单连线创建高保真交互原型,无需代码
- **智能功能**:自动布局、素材填充等AI辅助设计
- **超大协作**:支持最多500人同时在线协同工作

**适合场景**:
- ✅ 大型团队需要多人实时协作
- ✅ 产品经理需要快速制作可交互原型
- ✅ 字节系产品团队(原生支持)

**AI产品使用建议**:
- 用"原型文件"功能快速搭建AI工作流演示Demo
- 利用500人协同能力组织跨部门设计评审会

#### 5.3.3 Pixso

**官方定位**:AI原生UI设计工具,一键生成设计稿与代码(据Pixso官网)

**核心优势**:
- **AI生成**:识别页面风格,自动生成组件样式与配色方案
- **代码导出**:支持一键生成React/Vue代码,打通设计到开发
- **教育支持**:提供教育版与学生计划,适合教学使用
- **线框图资源**:内置大量优秀线框图模板,一键调用

**适合场景**:
- ✅ 初创团队需要快速从设计到代码
- ✅ 教育机构教学使用(有专门教育版)
- ✅ 需要频繁绘制线框图的产品经理

**AI产品使用建议**:
- 用Pixso AI生成多页面Demo,快速验证AI产品核心流程
- 利用内置线框图模板库,5分钟完成原型草图

#### 5.3.4 墨刀 (Modao)

**官方定位**:国产领先原型设计工具,2020年被万兴科技收购(据公开报道)

**核心优势**:
- **极低门槛**:内置数千APP/Web模板,几乎无学习成本
- **交互能力**:支持条件逻辑、变量等高级交互(媲美Axure)
- **AI Agent**:2026年推出AI自动生成原型功能
- **全端支持**:原型、设计、思维导图、流程图一体化

**适合场景**:
- ✅ 产品经理快速制作可点击原型
- ✅ 非设计专业人员制作演示Demo
- ✅ 需要复杂交互逻辑的原型设计

**AI产品使用建议**:
- 用墨刀快速搭建AI对话流程的多分支逻辑
- 利用AI Agent功能生成首版原型,再手动优化

#### 5.3.5 蓝湖 (Lanhu)

**官方定位**:产品设计协作平台,估值60亿人民币(据公开报道,2025年)

**核心特点**:
- **非设计工具**:需配合其他设计软件使用
- **协作管理**:设计稿上传、标注、版本控制
- **移动端查看**:随时查看高保真原型并评论
- **孵化MasterGo**:蓝湖公司开发了MasterGo设计工具

**适合场景**:
- ✅ 设计稿交付与版本管理
- ✅ 跨团队异步协作评审
- ✅ 已使用Sketch/Figma,需要协作平台

**AI产品使用建议**:
- 将AI产品设计稿上传蓝湖,方便异地团队评审
- 用蓝湖管理AI产品多版本迭代历史

### 实战洞察 💡

**中国工具vs海外工具的3个核心差异**:

1. **本土化优势**:
   - 微信/企业微信集成(即时设计、MasterGo支持小程序设计)
   - 中文字体库丰富(思源黑体、阿里巴巴普惠体等)
   - 符合国内网络环境(无需VPN,访问速度快)

2. **AI能力进化**:
   - Pixso/墨刀AI生成速度优于Figma AI(国内服务器优势)
   - MasterGo AI识别中文设计需求准确率更高

3. **成本结构**:
   - 免费版额度更大(墨刀/即时设计基础功能永久免费)
   - 企业版价格相比Figma低30-50%(据各平台官网定价,2026年)

**选型建议**:
- **纯设计团队** → Pixso/即时设计(功能完整,学习成本低)
- **产品经理主导** → 墨刀(快速原型,交互能力强)
- **大型企业** → MasterGo + 蓝湖(协作+交付管理)
- **教育场景** → Pixso教育版(有学生计划支持)

## 6. 保真度光谱 | The Fidelity Spectrum

### 6.1 5级保真度分类

设计过程不是"线框图→模型"的二元划分,而是**保真度连续光谱**:

```
低保真 ←────────────────────────────────→ 高保真
   │        │         │          │          │
   1        2         3          4          5
 手绘草图  数字线框   带样式     交互模型   功能原型
          图框图     线框图
```

**详细分级**:

| 级别 | 名称 | 特征 | 时间成本 | 典型工具 | 使用场景 |
|-----|------|------|---------|---------|---------|
| **Level 1** | 手绘草图 | 纸笔快速涂鸦,无规则 | 2-10分钟 | 纸、白板 | 头脑风暴、个人思考 |
| **Level 2** | 数字线框图 | 黑白灰,规整布局 | 5-30分钟 | 墨刀、Balsamiq | 需求评审、团队讨论 |
| **Level 3** | 带样式线框图 | 加入色彩与字体,无交互 | 30分钟-2小时 | 即时设计、Sketch | 中期设计评审 |
| **Level 4** | 静态模型 | 完整视觉设计,静态截图 | 1-4小时 | Pixso、MasterGo | 视觉确认、开发交付 |
| **Level 5** | 交互原型 | 可点击,模拟真实交互 | 4-8小时 | Figma、ProtoPie | 用户测试、投资人演示 |

### 6.2 AI产品保真度选择策略

**案例:AI写作助手从0到1的设计流程**

```
阶段1: 概念验证(Week 1)
├─ 工具: 白板手绘
├─ 保真度: Level 1
└─ 产出: "输入框+生成按钮+结果展示区"草图

阶段2: 功能评审(Week 2)
├─ 工具: 墨刀
├─ 保真度: Level 2
└─ 产出: 线框图展示"多轮对话"与"风格选择"逻辑

阶段3: 视觉方向确认(Week 3)
├─ 工具: 即时设计
├─ 保真度: Level 3
└─ 产出: 3种配色方案的带样式线框图

阶段4: 开发对接(Week 4)
├─ 工具: Pixso
├─ 保真度: Level 4
└─ 产出: 完整视觉稿+切图+标注

阶段5: 用户测试(Week 5)
├─ 工具: MasterGo交互原型
├─ 保真度: Level 5
└─ 产出: 可点击原型,模拟AI生成动画
```

**保真度跃迁的3个关键节点**:

1. **从Level 1到Level 2**:团队达成"功能范围共识"
   - 标志:产品需求文档(PRD)评审通过

2. **从Level 3到Level 4**:Stakeholder批准设计方向
   - 标志:CEO/设计总监签字确认视觉方案

3. **从Level 4到Level 5**:需要真实用户反馈
   - 标志:启动可用性测试(Usability Testing)

### 6.3 保真度陷阱与应对

**常见错误**:

❌ **过早高保真**:
- 表现:第一次需求会议就拿出精美模型
- 后果:团队纠结视觉细节,忽略功能逻辑缺陷
- 解决:严格执行"先低保真对齐,再高保真优化"

❌ **长期低保真**:
- 表现:接近开发阶段仍在用线框图
- 后果:工程师无法理解最终效果,返工率高
- 解决:在开发前2周必须交付Level 4模型

❌ **保真度不一致**:
- 表现:同一产品不同页面保真度差异大
- 后果:Stakeholder困惑,无法评估整体质量
- 解决:建立"设计检查清单",统一保真度标准

## 7. 敏捷开发中的应用

### 7.1 Sprint周期中的角色分工

**2周Sprint标准流程**:

```
Sprint Day 1-2: 需求澄清
├─ 工具: 线框图(墨刀)
├─ 参与者: PM + 设计师 + 工程师
└─ 产出: User Story对应的界面草图

Sprint Day 3-5: 设计执行
├─ 工具: 模型(Pixso/即时设计)
├─ 参与者: 设计师主导,PM Review
└─ 产出: 高保真视觉稿

Sprint Day 6-7: 开发对接
├─ 工具: 蓝湖(标注交付)
├─ 参与者: 设计师 + 工程师
└─ 产出: 开发所需切图与标注

Sprint Day 8-12: 开发实现
├─ 工具: 无(代码阶段)
├─ 参与者: 工程师
└─ 产出: 可运行的功能模块

Sprint Day 13-14: 测试与迭代
├─ 工具: 交互原型(对比实际产品)
├─ 参与者: QA + PM + 设计师
└─ 产出: Bug列表 + 下个Sprint需求
```

### 7.2 线框图在敏捷中的4个价值

1. **User Story可视化**:
   - 文本描述:`"作为AI客服用户,我希望查看历史对话记录"`
   - 线框图呈现:左侧折叠菜单展示"今日/本周/历史"分类

2. **跨角色共识工具**:
   - PM看到:功能完整性
   - 设计师看到:布局合理性
   - 工程师看到:实现复杂度

3. **快速迭代反馈**:
   - 每日站会展示线框图更新
   - 当天修改,次日评审

4. **技术债务可视化**:
   - 在线框图中标注`[技术限制:此功能V1.0不支持]`

### 7.3 模型在敏捷中的3个风险

⚠️ **风险1:过度设计**
- 表现:设计师追求视觉完美,延误开发
- 解决:设定"设计冻结日"(Design Freeze),之后只能微调

⚠️ **风险2:频繁返工**
- 表现:需求变更导致模型推倒重做
- 解决:前期用线框图充分验证,模型阶段限制需求变更

⚠️ **风险3:工程师误解**
- 表现:模型展示的交互效果,开发难以实现
- 解决:设计师在交付时附上"交互说明文档"

### 实战洞察 💡

**AI产品敏捷开发的线框图最佳实践**:

1. **用线框图标注"算法依赖"**:
   ```
   ┌────────────────────────────┐
   │ [推荐内容区]               │
   │ ⚠️ 依赖推荐算法v2.3        │ ← 明确标注技术依赖
   │ 预计延迟: <500ms           │
   │                            │
   │ [内容卡片1]                │
   │ [内容卡片2]                │
   │ [内容卡片3]                │
   └────────────────────────────┘
   ```

2. **用模型展示"AI能力边界"**:
   - 在模型中设计"生成失败"状态的友好提示
   - 避免过度承诺AI能力,设计降级方案

3. **建立"设计-算法"双周会**:
   - 每2周设计师与算法工程师同步线框图
   - 确保UI设计与模型能力匹配

## 核心术语表

| 中文术语 | 英文术语 | 定义 | AI产品应用 |
|---------|---------|------|-----------|
| **线框图** | Wireframe | 低保真结构图,聚焦布局与功能 | 验证AI对话流程逻辑 |
| **模型** | Mockup | 高保真视觉稿,展示品牌与美学 | 向投资人展示AI产品愿景 |
| **保真度** | Fidelity | 设计稿与最终产品的相似程度 | 选择合适保真度降低迭代成本 |
| **信息架构** | Information Architecture | 内容组织与导航结构 | 规划AI知识库的分类体系 |
| **用户流程** | User Flow | 用户完成任务的操作路径 | 设计AI写作助手的"需求→生成→编辑"流程 |
| **占位符** | Placeholder | 代表真实内容的临时元素 | 线框图中用`[AI生成内容区]`占位 |
| **高保真原型** | High-Fidelity Prototype | 接近真实产品的交互Demo | 模拟AI客服的多轮对话效果 |
| **设计系统** | Design System | 可复用的组件库与设计规范 | 统一AI产品矩阵的视觉风格 |
| **可用性测试** | Usability Testing | 真实用户操作产品的观察研究 | 测试AI推荐界面是否易懂 |
| **视觉层次** | Visual Hierarchy | 通过大小、颜色引导用户注意力 | 突出AI生成内容的关键信息 |
| **交互状态** | Interaction State | 元素的不同显示状态(默认/悬停/点击) | 设计AI按钮的加载动画 |
| **响应式设计** | Responsive Design | 适配不同屏幕尺寸的设计 | 确保AI工具在手机/平板可用 |
| **骨架屏** | Skeleton Screen | 内容加载时的占位动画 | AI生成时显示渐显文本框架 |

### 综合设计题

5. **开放题**:设计AI写作助手时,你发现用户在可用性测试中对"AI生成加载时间"表示不耐烦。请从线框图与模型两个层面,提出改进方案。

   <details>
   <summary>点击查看答案</summary>

   **线框图层面改进**:
   - 增加"流式输出"设计:逐字显示AI生成内容,而非等全部生成完毕
   - 标注加载状态层次:
     ```
     [AI思考中...] → [开始生成] → [生成第1段] → [生成完成]
     ```

   **模型层面改进**:
   - 设计骨架屏:显示文本框架渐显动画,营造"内容正在生成"感知
   - 添加进度提示:
     - 视觉:环形进度条 + 百分比(0% → 35% → 78% → 100%)
     - 文案:"正在构思开头..." → "正在撰写主体..." → "正在润色结尾..."
   - 提供候补交互:显示"推荐素材"卡片,让用户等待时有事可做

   **额外优化**:
   - 在线框图中标注性能指标:`目标加载时间 <3秒`
   - 在模型中设计"预加载"机制:用户输入时后台预生成常用模板
   </details>

### 批判性思考题

6. **辩论题**:有观点认为"AI产品迭代速度快,应该跳过线框图阶段直接做高保真原型"。你是否同意?请论述理由。

   <details>
   <summary>点击查看答案</summary>

   **不完全同意,需分场景讨论**:

   **支持跳过线框图的场景**:
   - ✅ **成熟产品微调**:已有设计系统,只需调整局部功能(如增加一个筛选按钮)
   - ✅ **紧急修复Bug**:用户反馈严重问题,需立即上线修复方案
   - ✅ **设计师经验丰富**:对AI产品交互模式理解深刻,可直接高保真输出

   **必须保留线框图的场景**:
   - ❌ **全新AI能力**:首次接入大模型,需线框图验证"提示词输入"与"结果展示"逻辑
   - ❌ **跨部门协作**:算法团队需要理解UI需求,线框图是最高效沟通工具
   - ❌ **复杂信息架构**:AI数据分析平台有10+功能模块,线框图帮助理清层级关系

   **推荐策略**:"混合保真度"——核心流程用线框图验证,次要页面直接高保真

   **案例**:
   - AI客服核心对话界面 → 先线框图对齐逻辑
   - AI客服设置页面 → 直接复用设计系统组件,跳过线框图
   </details>

### 练习1:线框图绘制(30分钟)

**任务**:为AI语音助手设计"语音转文字"功能的线框图

**要求**:
1. 使用墨刀或即时设计
2. 包含以下元素:
   - 录音按钮(默认/录音中/录音完成 三态)
   - 实时转写文本显示区
   - 编辑与分享按钮
   - 历史记录入口

- 布局清晰(30分):元素排列符合用户操作习惯
- 逻辑完整(40分):覆盖正常流程与异常情况(如网络中断)
- 标注清晰(30分):关键交互有文字说明

```

### 练习2:模型设计(90分钟)

**任务**:将练习1的线框图升级为高保真模型

**要求**:
1. 使用Pixso或即时设计
2. 应用完整视觉设计:
   - 品牌主色调(自选,如科技蓝#1E3A8A)
   - 图标库(可用iconfont.cn免费图标)
   - 真实文案(不使用"Lorem Ipsum")
3. 设计3个关键状态的模型:
   - 默认态(等待录音)
   - 录音中(显示波形动画)
   - 转写完成(显示文本结果)

**评分标准**:
- 视觉吸引力(30分):配色和谐,图标风格统一
- 品牌一致性(30分):符合AI产品的科技感定位
- 状态设计(40分):3个状态视觉区分明显,用户易理解

<details>
<summary>设计要点提示</summary>

**配色方案**:
- 主色调:科技蓝 `#1E3A8A` (按钮、标题)

- 按钮反馈:点击时缩放至95%

**注意事项**:
