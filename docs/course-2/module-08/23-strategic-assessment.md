# 23 | 战略评估 Strategic Assessment

## TL;DR (3分钟速读)

战略评估是通过系统性框架分析企业当前战略优劣势、外部机会威胁，并据此制定行动计划的过程。对于AI产品经理，战略评估不仅帮助理解公司大局，更是竞品分析、产品规划的核心工具。

**核心要点：**
- **SWOT分析**：内部优势/劣势 + 外部机会/威胁的经典四象限框架
- **2026中国AI实践**：技术快速迭代、监管政策频繁、用户需求多变
- **AI产品特殊性**：模型能力、数据资产、算力成本成为战略关键要素

**实战建议：**
1. **动态更新**：AI行业每季度进行一次SWOT评估（不是年度）
2. **量化指标**：用具体数据支撑分析（如模型参数量、推理速度、成本）
3. **交叉验证**：SWOT结论需与用户研究、竞品分析相互印证
4. **行动导向**：每个SWOT象限必须对应具体产品决策

## 1. 战略评估基础

### 1.1 什么是战略评估？

**战略评估（Strategic Assessment）** 是通过系统性方法，分析组织内部能力与外部环境，识别战略机会与风险，为战略制定与调整提供依据的过程。

**AI产品经理视角的战略评估**：
- **宏观层面**：理解公司在AI赛道的整体定位（如通用大模型 vs 垂直应用）
- **产品层面**：评估产品功能路线图的战略合理性
- **竞争层面**：识别竞品战略动向并提前布局应对
- **资源层面**：判断技术、团队、资金等资源配置是否支撑战略

**战略评估≠战略制定**：
- 战略评估：诊断现状、识别问题、发现机会（分析工具）
- 战略制定：基于评估结果，决定目标、路径、资源分配（决策过程）

### 1.2 为什么AI产品经理需要战略评估？

| **场景** | **战略评估作用** | **AI产品示例** |
|---------|----------------|---------------|
| **产品规划** | 识别最有价值的功能方向 | 豆包评估：多模态能力 vs 推理能力，哪个是核心优势？ |
| **竞品应对** | 理解对手战略意图，避免被动跟随 | Kimi推出长文本，是战略核心还是防御性功能？ |
| **资源争取** | 向上争取预算/人力时，用战略逻辑说服决策层 | "根据SWOT分析，我们应优先投入XX功能，因为..." |
| **职业发展** | 展现战略思维，从执行型PM升级为战略型PM | 能从战略高度解释产品决策的PM更受重视 |

## 2. SWOT分析框架

### 2.1 SWOT四象限

**SWOT** = Strengths（优势） + Weaknesses（劣势） + Opportunities（机会） + Threats（威胁）

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
           │    内部因素    │   外部因素
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  正面影响  │  S 优势       │  O 机会
           │  Strengths    │  Opportunities
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  负面影响  │  W 劣势       │  T 威胁
           │  Weaknesses   │  Threats
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**内部 vs 外部判断标准**：
- **内部因素**：公司可控、可直接改变（如团队技能、技术积累、品牌影响力）
- **外部因素**：公司不可控、只能适应（如政策法规、市场趋势、竞争格局）

**正面 vs 负面判断标准**：
- **正面影响**：有利于实现战略目标
- **负面影响**：阻碍实现战略目标

### 2.2 SWOT四象限详解

#### S - Strengths（优势）
**定义**：组织内部相对竞争对手的优越之处

**AI产品常见优势维度**：
- **技术能力**：模型性能、推理速度、多模态能力
- **数据资产**：训练数据规模、数据质量、独家数据源
- **团队实力**：研发团队规模、顶尖科学家数量
- **品牌影响力**：用户认知度、行业口碑
- **生态资源**：合作伙伴、API集成、开发者社区

**优势识别方法**：
1. **对比法**：与竞品横向对比，找出领先指标
2. **用户反馈**：用户最常夸奖的产品特性
3. **数据验证**：用量化指标证明优势（如日活、留存率）

**2026中国AI产品优势示例**：
- **字节跳动豆包**：算法推荐能力（基于抖音/今日头条积累）
- **百度文心一言**：中文语料积累、搜索数据优势
- **阿里通义千问**：电商场景数据、云计算基础设施

#### W - Weaknesses（劣势）
**定义**：组织内部相对竞争对手的不足之处

**AI产品常见劣势维度**：
- **技术短板**：模型能力不足、推理成本过高
- **资源限制**：算力资源紧张、资金压力
- **品牌认知**：新产品知名度低、用户信任不足
- **团队缺口**：缺少关键岗位人才
- **合规风险**：数据安全、隐私保护不足

**劣势识别方法**：
1. **对标法**：与行业标杆对比，找出差距
2. **用户投诉**：用户最常抱怨的产品问题
3. **内部反思**：团队成员匿名反馈产品痛点

**2026中国AI产品劣势示例**：
- **初创公司Kimi**：算力成本高（依赖云服务）、品牌影响力不足
- **传统企业AI产品**：技术团队积累弱、产品迭代速度慢

#### O - Opportunities（机会）
**定义**：外部环境中有利于组织发展的趋势或事件

**AI产品常见机会维度**：
- **政策红利**：国家支持AI发展、行业补贴政策
- **技术突破**：开源模型进步、推理成本下降
- **市场需求**：新兴场景爆发（如AI客服、AI教育）
- **竞争格局变化**：主要竞品出现战略失误
- **用户习惯改变**：用户接受AI产品的门槛降低

**机会识别方法**：
1. **趋势分析**：阅读行业报告、专家预测
2. **政策跟踪**：关注国家/地方AI扶持政策
3. **用户需求挖掘**：从用户访谈中发现未被满足的痛点

**2026中国AI产品机会示例**：
- **AI Agent应用爆发**：从单轮对话到多步骤任务执行
- **端侧AI普及**：手机、IoT设备AI算力提升，降低云端依赖
- **行业垂直化**：医疗、法律、教育等垂直领域对AI需求激增

#### T - Threats（威胁）
**定义**：外部环境中不利于组织发展的趋势或事件

**AI产品常见威胁维度**：
- **监管政策**：算法备案、内容审核趋严
- **技术风险**：开源模型快速迭代，商业模型优势被削弱
- **竞争加剧**：巨头降价、新玩家入场
- **用户信任危机**：AI幻觉、隐私泄露事件频发
- **成本压力**：算力成本、人力成本持续上升

**威胁识别方法**：
1. **风险扫描**：定期评估政策、技术、竞争风险
2. **竞品监控**：跟踪竞品动态（如降价、新功能发布）
3. **舆情分析**：监控行业负面新闻、用户信任度变化

**2026中国AI产品威胁示例**：
- **监管趋严**：生成式AI服务需备案、内容安全审核加强
- **开源冲击**：Llama 4、Qwen等开源模型性能逼近商业模型
- **价格战**：巨头通过降价挤压中小玩家生存空间

### 2.3 SWOT交叉分析（TOWS矩阵）

单独列举SWOT不够，需进行**交叉分析**，形成具体战略：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
              │  S 优势          │  W 劣势
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  O 机会       │  SO策略          │  WO策略
              │ （增长型战略）    │ （扭转型战略）
              │ 利用优势抓住机会  │ 弥补劣势抓住机会
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  T 威胁       │  ST策略          │  WT策略
              │ （多元化战略）    │ （防御型战略）
              │ 利用优势规避威胁  │ 减少劣势规避威胁
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**四大战略方向**：
- **SO策略**（增长型）：优势+机会 → 最理想，快速扩张
- **WO策略**（扭转型）：劣势+机会 → 补短板，抓住窗口期
- **ST策略**（多元化）：优势+威胁 → 避开威胁，发挥优势
- **WT策略**（防御型）：劣势+威胁 → 最危险，考虑战略收缩

## 3. AI产品的战略评估特殊性

### 3.1 AI产品vs传统软件：SWOT差异

| **维度** | **传统软件** | **AI产品** |
|---------|------------|----------|
| **核心优势来源** | 产品功能、用户体验 | 模型能力、数据资产、算力效率 |
| **劣势表现** | 功能缺失、UI体验差 | 模型幻觉、推理成本高、合规风险 |
| **机会窗口** | 用户需求变化、市场空白 | 技术突破（如GPT-5）、开源生态、政策红利 |
| **威胁来源** | 竞品功能、商业模式 | 开源模型冲击、监管政策、算力成本上涨 |
| **评估频率** | 年度/半年度 | 季度/月度（AI迭代速度快） |

### 3.2 AI产品SWOT关键评估维度

#### **Strengths（优势）评估清单**
- [ ] **模型能力**：在核心任务上的表现（如准确率、推理速度）
- [ ] **数据优势**：独有数据源、数据规模、数据质量
- [ ] **算力资源**：自有算力 vs 云服务，成本优势
- [ ] **技术团队**：顶尖研究员、工程能力
- [ ] **产品体验**：界面易用性、响应速度、多模态支持
- [ ] **生态整合**：API开放、开发者社区、合作伙伴

#### **Weaknesses（劣势）评估清单**
- [ ] **模型短板**：幻觉率高、特定任务表现差
- [ ] **成本压力**：推理成本高、难以盈利
- [ ] **合规风险**：内容安全、隐私保护不足
- [ ] **品牌认知**：用户不信任、市场份额低
- [ ] **资源限制**：算力紧张、资金链压力
- [ ] **人才缺口**：缺少关键岗位（如RLHF专家）

#### **Opportunities（机会）评估清单**
- [ ] **技术突破**：新模型架构、开源工具进步
- [ ] **政策支持**：国家AI战略、地方补贴
- [ ] **市场需求**：新兴场景（如AI Agent、AI教育）
- [ ] **竞争格局**：竞品战略失误、市场洗牌
- [ ] **用户教育**：用户对AI接受度提升
- [ ] **成本下降**：算力成本、推理成本降低

#### **Threats（威胁）评估清单**
- [ ] **监管趋严**：算法备案、内容审核、数据安全
- [ ] **开源冲击**：开源模型性能逼近商业模型
- [ ] **价格战**：巨头降价、免费策略
- [ ] **信任危机**：AI幻觉事件、隐私泄露丑闻
- [ ] **技术风险**：核心技术被竞品超越
- [ ] **成本上涨**：算力、人力成本持续上升

## 4. 2026中国AI产品SWOT案例

### 4.1 案例：字节跳动豆包SWOT分析

**背景**：豆包是字节跳动推出的AI助手，对标ChatGPT/Kimi，2026年1月MAU达8500万。

#### **S - Strengths（优势）**

| **优势** | **具体表现** | **数据支撑** |
|---------|------------|------------|
| 算法推荐能力 | 基于抖音/今日头条积累的推荐算法，精准理解用户意图 | 抖音日活超7亿，积累大量用户行为数据 |
| 流量入口 | 抖音、今日头条等亿级流量产品内嵌豆包 | 豆包MAU达8500万（2026年1月） |
| 内容生态 | 擅长短视频脚本、社交文案生成 | 抖音创作者使用豆包生成视频脚本 |
| 资金充足 | 字节跳动年收入超1200亿美元，AI投入不设上限 | 2025年AI算力投入超100亿元 |

#### **W - Weaknesses（劣势）**

| **劣势** | **具体表现** | **影响** |
|---------|------------|---------|
| 长文本能力弱 | 单次对话上下文窗口小于Kimi（支持32K vs Kimi 200K） | 无法处理超长文档分析任务 |
| 专业领域深度不足 | 在法律、医疗等垂直领域表现不如专用模型 | 难以进入B端专业市场 |
| 品牌信任度 | 部分用户担心字节跳动滥用数据 | 企业用户对数据安全顾虑 |
| 国际化受限 | 主要聚焦中国市场，海外拓展缓慢 | 无法与OpenAI等全球竞争 |

#### **O - Opportunities（机会）**

| **机会** | **具体描述** | **豆包如何抓住** |
|---------|------------|---------------|
| AI Agent爆发 | 从单轮对话→多步骤任务执行（如自动剪辑视频） | 结合剪映、抖音生态，打造视频创作AI Agent |
| 短视频AI需求 | 创作者对AI辅助创作需求激增 | 深度集成抖音，提供脚本生成、配音、字幕 |
| 端侧AI普及 | 手机AI算力提升，本地推理成为可能 | 与OPPO、vivo合作，推出端侧豆包 |
| 政策红利 | 国家支持生成式AI发展，算力补贴 | 申请国家AI专项资金支持 |

#### **T - Threats（威胁）**

| **威胁** | **具体描述** | **潜在影响** |
|---------|------------|------------|
| 监管趋严 | 生成式AI内容审核要求提高 | 需增加审核成本，响应速度变慢 |
| 开源模型冲击 | Qwen、GLM等开源模型免费可用 | 部分用户转向自建开源方案 |
| Kimi长文本优势 | Kimi在长文本领域建立用户心智 | 豆包在文档分析场景失去竞争力 |
| 价格战 | 腾讯元宝、百度文心降价 | 被迫降低API定价，利润空间缩小 |

#### **TOWS交叉分析：豆包的战略方向**

| **交叉策略** | **具体行动** |
|------------|------------|
| **SO策略（增长型）** | 利用流量优势（S）+ 抓住短视频AI需求（O）→ **强化视频创作AI Agent** |
| **WO策略（扭转型）** | 弥补长文本弱势（W）+ 抓住端侧AI机会（O）→ **推出端侧长文本模型** |
| **ST策略（多元化）** | 利用内容生态优势（S）+ 应对开源冲击（T）→ **打造垂直场景闭环（如电商文案）** |
| **WT策略（防御型）** | 减少国际化弱势（W）+ 应对监管趋严（T）→ **聚焦中国市场，加强合规** |

**豆包2026年战略决策**：
1. **核心战略**（SO）：深度绑定抖音生态，成为视频创作者首选AI工具
2. **补短板**（WO）：与手机厂商合作，推出端侧长文本能力
3. **差异化**（ST）：避开通用对话赛道，专注短视频/电商内容生成
4. **风险控制**（WT）：暂缓海外扩张，优先通过国内合规审核

### 4.2 案例：Kimi SWOT分析（月之暗面）

**背景**：Kimi是月之暗面推出的AI助手,以"超长文本处理"为核心卖点,2026年1月MAU达2000万。

#### **S - Strengths（优势）**
- 长文本能力：支持200K token上下文窗口（行业领先）
- 品牌定位清晰：用户心智="长文本就用Kimi"
- 口碑传播：知乎、小红书用户自发推荐
- 技术团队：月之暗面创始团队来自清华NLP实验室

#### **W - Weaknesses（劣势）**
- 算力成本高：长文本推理成本是短文本的5-10倍
- 流量入口少：缺少像抖音这样的亿级流量产品
- 功能单一：除长文本外,其他功能无明显优势
- 盈利压力：免费用户多,付费转化率低

#### **O - Opportunities（机会）**
- 专业市场需求：律师、研究员、分析师等需要处理长文档
- B端市场：企业知识库、合同审核等场景
- 技术突破：推理成本下降,长文本门槛降低
- 融资环境改善：AI创业公司估值回升

#### **T - Threats（威胁）**
- 巨头追赶：豆包、文心一言开始强化长文本能力
- 开源替代：Qwen-Long等开源长文本模型出现
- 成本压力：算力成本居高不下,难以盈利
- 用户期望过高：部分用户认为长文本=万能,实际体验不符

#### **TOWS交叉分析：Kimi的战略方向**

| **交叉策略** | **具体行动** |
|------------|------------|
| **SO策略** | 利用长文本优势 + 专业市场需求 → **进军B端法律/金融市场** |
| **WO策略** | 弥补流量劣势 + B端机会 → **与企业微信/飞书集成,获取企业流量** |
| **ST策略** | 利用技术优势 + 应对巨头威胁 → **开放API,成为长文本能力供应商** |
| **WT策略** | 降低成本压力 + 应对开源冲击 → **推出付费订阅,筛选高价值用户** |

**Kimi 2026年战略决策**：
1. **核心战略**（SO）：从C端工具转向B端SaaS（如法律文书分析平台）
2. **生态合作**（WO）：与钉钉/飞书深度集成，获取企业用户
3. **技术开放**（ST）：推出API服务，降低巨头威胁
4. **商业化加速**（WT）：推出Pro版本（200元/月），筛选付费用户

### 5.1 战略评估标准流程（6步法）

```
┌─────────────┐

│ 2. 收集信息  │  内部数据 + 外部信息（竞品/行业报告）
└──────┬──────┘
       │
┌──────▼──────┐
│ 3. SWOT分析 │  填充四象限，量化评估
└──────┬──────┘
       │
┌──────▼──────┐
│ 4. 交叉分析  │  TOWS矩阵，形成战略方向
└──────┬──────┘
```

### 5.2 Step 1: 明确评估目标

**常见评估目标**：
- **产品规划**：下一季度功能优先级排序
- **竞品应对**：制定针对性竞争策略
- **融资准备**：向投资人展示战略清晰度
- **团队对齐**：统一团队对战略方向的认知

**目标决定评估深度**：
- 产品规划：聚焦产品层面SWOT即可
- 融资准备：需做公司整体SWOT + 行业分析

### 5.3 Step 2: 收集信息

**内部信息收集**：
- **产品数据**：DAU、留存率、付费转化率、核心功能使用率
- **技术指标**：模型性能、推理成本、服务稳定性
- **团队访谈**：研发/运营/销售对产品的反馈
- **用户反馈**：客服工单、用户调研、NPS评分

**外部信息收集**：
- **竞品动态**：竞品发布、融资、用户评价（七麦数据、App Annie）
- **行业报告**：艾瑞咨询、易观分析、IDC的AI行业报告
- **政策法规**：国家AI政策、算法备案要求
- **技术趋势**：arXiv论文、开源模型进展

**中国AI产品信息源推荐**：
| **类型** | **工具/平台** | **用途** |
|---------|-------------|---------|
| 竞品监控 | 七麦数据、App Growing | 竞品APP排名、用户评价 |
| 行业报告 | 艾瑞咨询、易观分析 | 市场规模、增长趋势 |
| 政策跟踪 | 国家网信办官网、工信部 | AI监管政策更新 |
| 技术趋势 | arXiv、HuggingFace | 最新模型、开源工具 |
| 用户舆情 | 知乎、小红书、微博 | 用户真实评价 |

### 5.4 Step 3: SWOT分析（填充四象限）

**操作建议**：
1. **头脑风暴**：团队一起列举SWOT项（每人准备5条）
2. **去重合并**：将相似项合并（如"模型速度快""推理效率高"合并为"推理性能优势"）
3. **优先级排序**：每个象限保留最重要的5-7项
4. **量化验证**：用数据支撑每一项（如"用户增长快" → "MAU月增20%"）

**SWOT模板示例**：

| **S 优势** | **W 劣势** |
|-----------|-----------|
| 1. 模型推理速度快（平均响应<2秒） | 1. 长文本能力弱（仅支持32K token） |
| 2. 品牌知名度高（MAU 8500万） | 2. B端市场渗透率低（<5%企业用户） |
| 3. 算法推荐精准（CTR 15%,行业平均10%） | 3. 算力成本高（单次推理成本0.05元） |
| ... | ... |

| **O 机会** | **T 威胁** |
|-----------|-----------|
| 1. AI Agent应用爆发（预计2026年市场规模500亿） | 1. 开源模型Qwen性能逼近（差距<5%） |
| 2. 短视频AI创作需求激增（抖音创作者2亿+） | 2. 监管趋严（算法备案周期延长至3个月） |
| 3. 端侧AI算力提升（旗舰手机支持70B模型） | 3. 价格战加剧（腾讯元宝免费策略） |
| ... | ... |

### 5.5 Step 4: TOWS交叉分析

**操作步骤**：
1. **SO配对**：找出"哪些优势可以抓住哪些机会"
2. **WO配对**：找出"补哪些劣势可以抓住机会"
3. **ST配对**：找出"哪些优势可以规避威胁"
4. **WT配对**：找出"如何减少劣势并规避威胁"

**豆包TOWS矩阵示例**：

|  | **S 优势** | **W 劣势** |
|--|-----------|-----------|
| **O 机会** | SO: 流量优势 + 短视频AI需求 = **视频创作AI Agent** | WO: 长文本弱 + 端侧AI = **端侧长文本模型** |
| **T 威胁** | ST: 内容生态 + 开源冲击 = **垂直场景闭环** | WT: 国际化弱 + 监管严 = **聚焦中国市场** |

### 5.6 Step 5: 决策制定

**从SWOT到产品Roadmap**：
1. **SO策略 → OKR目标**：如"Q2推出视频创作AI Agent，MAU增至1000万"
2. **WO策略 → 功能优先级**：如"Q2优先开发长文本能力，追平Kimi"
3. **ST策略 → 差异化定位**：如"放弃通用对话，专注电商文案生成"
4. **WT策略 → 风险控制**：如"暂停国际化，优先通过国内合规"

**决策示例（豆包2026 Q2 Roadmap）**：

| **战略方向** | **具体产品决策** | **成功指标** |
|------------|---------------|------------|
| SO: 视频创作AI Agent | 集成剪映，支持"一键生成60秒带货视频" | 日均生成视频100万条 |
| WO: 端侧长文本 | 与OPPO合作，推出手机端32K长文本 | 端侧用户占比达20% |
| ST: 垂直场景闭环 | 推出"电商文案助手"，与淘宝/抖音小店打通 | 电商创作者付费率10% |
| WT: 聚焦中国市场 | 完成算法备案，通过内容安全审核 | 备案通过率100% |

### 5.7 Step 6: 跟踪执行

**季度复盘机制**：
- **Q1结束**：回顾SWOT评估，检查哪些假设成立/失效
- **Q2开始**：更新SWOT（新增威胁、机会变化等）
- **动态调整**：根据实际执行情况修正战略

### 6.1 SWOT之外的战略评估工具

| **工具** | **核心用途** | **适用场景** |
|---------|------------|------------|

### 6.2 工具选择指南

**SWOT vs PEST**：
- **SWOT**：关注企业自身，适合产品规划、竞品应对
- **PEST**：关注宏观环境，适合进入新市场、长期战略

**SWOT vs 波特五力**：
- **SWOT**：评估当前状态（现在有什么优劣势）
- **波特五力**：评估行业结构（行业是否值得进入）

**SWOT vs 价值链**：
- **SWOT**：全面评估（内外部、正负面）
- **价值链**：聚焦内部运营（研发/生产/营销各环节）

**AI产品PM常用组合**：
1. **SWOT + PEST**：全面评估宏观环境 + 企业定位
2. **SWOT + 波特五力**：评估行业吸引力 + 企业优劣势
3. **SWOT + 价值链**：找出核心优势来源

### 6.3 PEST分析（AI产品视角）

**PEST** = Political（政治）+ Economic（经济）+ Social（社会）+ Technological（技术）

**2026中国AI产品PEST分析示例**：

| **维度** | **关键因素** | **影响** |
|---------|------------|---------|
| **P 政治** | 国家支持AI发展，但监管趋严（算法备案、内容审核） | 正面：政策红利；负面：合规成本上升 |
| **E 经济** | AI投资趋于理性，融资环境收紧 | 初创公司融资难度增加 |
| **S 社会** | 用户对AI接受度提升，但担心隐私泄露 | 正面：市场需求增长；负面：需加强隐私保护 |
| **T 技术** | 开源模型快速迭代，算力成本下降 | 正面：技术门槛降低；负面：商业模型优势削弱 |

9. 游戏NPC对话（与游戏公司合作）
10. AI绘画功能（对标Midjourney）

**SWOT优先级评分**：

| **功能** | **SWOT类型** | **战略得分** | **资源投入** | **优先级** |
|---------|------------|------------|------------|-----------|

1. **视频创作AI Agent**（SO战略）- 团队A全力投入

- 放弃WT功能（海外/绘画）：避开劣势领域，聚焦核心战场

## 9. 战略评估常见误区

### 9.1 误区1：SWOT沦为"列举清单"

**错误做法**：
- 机械列举30条优势、20条劣势，没有优先级
- 没有量化验证，全是主观判断（如"我们技术很强"）

**正确做法**：
- 每个象限保留最重要的5-7项
- 用数据支撑每一项（如"推理速度快" → "平均响应1.8秒，行业平均3秒"）
- 团队投票排序，聚焦核心SWOT

### 9.2 误区2：SWOT填完就结束

**错误做法**：
- 只做SWOT分析，不做TOWS交叉
- 没有转化为具体产品决策

**正确做法**：
- 必须做TOWS交叉分析，形成战略方向
- 每个战略方向对应具体OKR/Roadmap

### 9.3 误区3：内外部因素混淆

**错误做法**：
- 将"竞品降价"列为劣势（应为威胁）
- 将"政策支持"列为优势（应为机会）

**正确做法**：
- **内部因素**（S/W）：公司可控、可直接改变
- **外部因素**（O/T）：公司不可控、只能适应

**判断技巧**：问自己"公司能否直接改变这一项？"
- 能改变 → 内部（S/W）
- 不能改变 → 外部（O/T）

### 9.4 误区4：静态评估，从不更新

**错误做法**：
- 一年做一次SWOT，中间从不更新
- AI行业变化快，半年前的SWOT已过时

**正确做法**：
- AI产品：每季度更新SWOT
- 重大事件后立即更新（如竞品发布、政策变化）
- 建立SWOT追踪机制（见5.7节）

### 9.5 误区5：优势≠竞争优势

**错误做法**：
- 列举"我们有AI团队""我们有云服务"（行业标配，不是优势）

**正确做法**：
- 优势必须是**相对竞争对手的优越之处**
- 对标竞品，找出领先指标（如"推理速度比Kimi快30%"）

## 💡 实战洞察

### 洞察1：SWOT不是"写作业"，是"找方向"

很多PM把SWOT当作"必须填完的表格"，机械列举30条优势、20条劣势，最后束之高阁。**真正的战略评估应该是动态的决策工具**：

- **每季度更新**：AI行业变化快，半年前的SWOT已过时
- **量化验证**：每一项都要有数据支撑（不是"感觉技术很强"，而是"推理速度1.8秒 vs 行业平均3秒"）
- **行动导向**：SWOT的终点是产品决策，不是分析报告

**实战建议**：设置"SWOT季度会"，团队一起回顾上季度假设是否成立，更新下季度SWOT。

### 洞察2：AI产品的"优势"会快速贬值

传统软件的优势可以持续数年（如微软Office的用户习惯），但AI产品的优势可能3个月就被超越：

- **技术优势**：今天领先的模型能力，3个月后可能被开源模型追平
- **成本优势**：算力成本快速下降，今天的成本优势明天可能消失
- **功能优势**：竞品抄袭速度极快（Kimi长文本→豆包3个月推出）

**实战建议**：不要把单一技术指标当作护城河，而要构建**系统性优势**（如抖音流量+算法+内容生态）。

### 洞察3：中国AI市场的"威胁"具有特殊性

相比国际市场，中国AI产品面临独特威胁：

- **监管不确定性**：算法备案、内容审核政策频繁调整
- **开源生态强大**：Qwen、GLM等国产开源模型快速迭代
- **价格战激烈**：巨头通过免费/低价策略挤压中小玩家

**实战建议**：在SWOT评估中，"威胁"部分要特别关注**政策风险**和**开源冲击**，提前准备预案。

## 📚 核心术语表

| **英文术语** | **中文翻译** | **定义** |
|------------|------------|---------|
| **Strategic Assessment** | 战略评估 | 通过系统性方法分析组织内外部环境，为战略决策提供依据 |
| **SWOT Analysis** | SWOT分析 | Strengths/Weaknesses/Opportunities/Threats四象限框架 |
| **TOWS Matrix** | TOWS矩阵 | SWOT交叉分析工具，形成SO/WO/ST/WT四大战略方向 |
| **Internal Factors** | 内部因素 | 组织可控、可直接改变的因素（如技术能力、团队） |
| **External Factors** | 外部因素 | 组织不可控、只能适应的因素（如政策、竞争格局） |
| **Competitive Advantage** | 竞争优势 | 相对竞争对手的优越之处（不是绝对优势） |
| **Strategic Gap** | 战略差距 | 当前状态与目标状态之间的差距 |
| **PEST Analysis** | PEST分析 | Political/Economic/Social/Technological宏观环境分析 |
| **Porter's Five Forces** | 波特五力 | 评估行业竞争格局的框架 |
| **Value Chain Analysis** | 价值链分析 | 评估企业内部各环节价值创造能力 |

## 🎯 实战练习

### 练习1：为你负责的产品做SWOT分析（30分钟）

**步骤**：
1. 选择一款AI产品（豆包/Kimi/文心一言等）
2. 收集最新数据（MAU、功能、用户评价）
3. 填充SWOT四象限（每个象限5-7项）
4. 完成TOWS交叉分析
5. 提出3个具体产品决策

**输出**：一份完整的SWOT分析报告（参考10.1节模板）

### 练习2：竞品战略评估对比（45分钟）

**任务**：对比分析豆包 vs Kimi的SWOT，回答：
1. 两者最大的战略差异是什么？
2. 如果你是Kimi PM，如何应对豆包的流量优势？
3. 如果你是豆包PM，如何应对Kimi的长文本优势？

**输出**：一份竞品战略对比报告（包含双方SWOT + 应对策略）

### 练习3：SWOT动态追踪（持续）

**任务**：建立SWOT季度追踪机制

4. 季度末回顾：哪些假设成立？哪些失效？

**输出**：一份持续更新的SWOT追踪表
