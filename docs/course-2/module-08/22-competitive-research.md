# 22 | 研究竞争对手 Researching Competitors

## TL;DR (3分钟速读)

竞品分析是通过系统性研究竞争对手的产品、策略、优劣势，来指导自己产品决策的过程。

**核心要点**：
- **定义竞争对手**：直接竞争对手（同功能同用户）、间接竞争对手（替代解决方案）、潜在竞争对手（可能进入的玩家）
- **4个层次分析**：产品功能 → 商业模式 → 技术架构 → 战略定位
- **信息来源**：公开渠道（官网/App/社交媒体）、第三方数据（艾瑞/易观/Sensor Tower）、一线情报（用户访谈/行业峰会）、逆向工程（产品拆解）
- **2026中国AI产品竞争格局**：大模型赛道"百模大战"（通义/文心/豆包/Kimi/智谱等20+玩家）、AIGC工具赛道（文生图/文生视频/AI设计）、垂直场景AI（法律/医疗/教育）

**关键工具**：
```
信息收集工具（中国版）：
- 公司信息：天眼查、企查查
- 流量数据：七麦数据（App）、5118（Web）、艾瑞咨询
- 用户评价：应用商店评论、小红书、知乎、微博
- 技术架构：产品拆解、API监测
- 财务数据：IT桔子、36氪、企名片
```

**实战建议**：
1. **定期更新**：AI产品迭代快，每月更新竞品信息（不是季度）
2. **聚焦关键竞品**：深度分析3-5个核心竞品，而非泛泛研究20个
3. **用户视角**：亲自使用竞品（注册账号、付费体验），而非只看报告
4. **合法合规**：绝不窃取商业机密、源代码、用户数据

## 1. 什么是竞品分析

### 1.1 定义

**竞品分析 (Competitive Analysis)** = 系统性研究竞争对手的**产品、策略、优劣势**，以指导自己产品决策的过程。

**核心目标**：
1. **知己知彼**：了解竞争对手在做什么、做得如何
2. **寻找机会**：发现竞品的弱点，自己的差异化空间
3. **学习借鉴**：吸收竞品的优秀实践
4. **预判趋势**：通过竞品动态，预测行业走向

### 1.2 为什么竞品分析对AI产品特别重要？

```
AI产品 vs 传统软件：竞品分析的差异

传统软件（如Excel）：
  ├─ 竞争格局：稳定（Google Sheets、WPS表格）
  ├─ 迭代周期：年度大版本更新
  ├─ 技术门槛：高（10年开发经验）
  └─ 竞品分析频率：每季度1次

AI产品（如AI对话）：
  ├─ 竞争格局：极不稳定（每月新玩家、每周新功能）
  ├─ 迭代周期：周更新、日更新
  ├─ 技术门槛：相对低（开源模型+API调用）
  └─ 竞品分析频率：每月1次甚至每周1次 ⚠️

AI产品的特殊挑战：
1. 新玩家频繁出现（2024-2026年中国大模型从5个→20+个）
2. 功能快速同质化（竞品上线新功能，1周内被抄袭）
3. 技术突破难预测（GPT-5发布可能颠覆整个格局）
4. 用户迁移成本低（切换AI产品只需10分钟）
```

**行业观察**：
- AI产品PM通常每周投入显著时间做竞品分析（vs 传统软件PM每季度分析）
- 头部AI产品团队普遍建立竞品监测系统，自动抓取更新（新功能、定价变化、用户评价）

## 2. 识别竞争对手：3类竞争对手

### 2.1 竞争对手的3个层次

```
竞争对手的3圈层模型

           ┌──────────────────────────────┐
           │  潜在竞争对手 (Potential)      │
           │  可能进入市场的玩家            │
           │  ┌────────────────────────┐  │
           │  │ 间接竞争对手 (Indirect) │  │
           │  │ 替代解决方案            │  │
           │  │ ┌──────────────────┐  │  │
           │  │ │ 直接竞争对手      │  │  │
           │  │ │ (Direct)         │  │  │
           │  │ │ 同功能同用户      │  │  │
           │  │ └──────────────────┘  │  │
           │  └────────────────────────┘  │
           └──────────────────────────────┘
```

#### 层次1：直接竞争对手 (Direct Competitors)

**定义**：与你的产品**功能相同、目标用户相同**的竞品

**识别标准**：
- ✅ 功能重叠度>70%
- ✅ 目标用户群体相同
- ✅ 商业模式相似
- ✅ 用户在两者之间做选择

**示例（教学场景）：豆包的直接竞争对手分析**

| 竞品 | 功能重叠度（估算） | 目标用户 | 为什么是直接竞争 |
|------|----------|---------|----------------|
| **文心一言** | 高 | 通用AI对话用户 | 百度出品，功能几乎一致，用户在选择"用豆包还是文心" |
| **通义千问** | 高 | 通用AI对话用户 | 阿里出品，主打工作提效，与豆包重叠 |
| **Kimi** | 中 | 专业用户（超长文本） | 虽然定位垂直，但核心功能仍是AI对话，有重叠 |
| **讯飞星火** | 高 | 通用AI对话用户 | 科大讯飞出品，语音优势，但整体功能相似 |

**优先级**：🔥🔥🔥 **最高**（必须深度分析）

---

#### 层次2：间接竞争对手 (Indirect Competitors)

**定义**：功能不同，但**解决同样用户需求**的替代方案

**识别标准**：
- ✅ 解决同样的"Job to Be Done"
- ✅ 用户场景重叠
- ❌ 但产品形态不同

**示例：豆包的间接竞争对手**

| 竞品 | 用户需求 | 为什么是间接竞争 | 威胁程度 |
|------|---------|----------------|---------|
| **Notion AI** | 工作文档撰写 | 用户本来想用豆包写文档，但Notion AI集成在编辑器里更方便 | 中 |
| **飞书智能伙伴** | 企业办公提效 | 企业用户用飞书智能伙伴处理工作任务，就不用豆包了 | 高 |
| **GitHub Copilot** | 代码编写 | 程序员用Copilot写代码，就不用豆包了 | 中 |
| **ChatGPT** | 通用AI对话 | 虽然在中国需要翻墙，但部分高端用户仍使用（English能力强） | 高 |

**关键洞察**：间接竞争对手往往被忽视，但可能**抢走你最有价值的用户**

**案例**：
```
基于行业观察：
  - AI对话产品在程序员群体中面临间接竞品挑战
  - GitHub Copilot等IDE集成工具因便捷性抢占用户
  - 用户反馈：切换窗口影响工作流

应对策略：
  ✅ 推出编辑器插件（VS Code/JetBrains等）
  ✅ 强化开发者场景集成能力
```

**优先级**：🔥🔥 **高**（需要关注，尤其是抢走高价值用户的竞品）

---

#### 层次3：潜在竞争对手 (Potential Competitors)

**定义**：现在不是竞品，但**未来可能进入市场**的玩家

**识别标准**：
- 有技术能力
- 有用户基础
- 有资金实力
- 有进入动机

**示例：豆包的潜在竞争对手（2026年预判）**

| 潜在竞品 | 当前业务 | 为什么可能进入 | 进入概率 |
|---------|---------|--------------|---------|
| **微信** | 社交 | 10亿用户，可以在微信内置AI对话（类似微信AI） | 80% |
| **滴滴** | 出行 | 有AI技术积累，可能推出"滴滴AI助手"（车上闲聊场景） | 30% |
| **拼多多** | 电商 | 可能推出"AI购物助手"（帮用户找商品、比价） | 50% |
| **美团** | 本地生活 | 可能推出"AI生活助手"（订餐、订酒店、规划行程） | 60% |

**关键动作**：
1. **监测信号**：关注潜在竞品的招聘（如"美团招聘大模型算法工程师"）
2. **提前布局**：在潜在竞品进入前，抢占核心场景（如豆包提前做"购物助手"功能）

**优先级**：🔥 **中**（长期关注，但不必立即深度分析）

### 2.2 如何识别竞争对手？4个方法

#### 方法1：用户视角（最准确）

**问题**："如果用户不用你的产品，他会用什么？"

**步骤**：
1. 访谈10-20个真实用户/潜在用户
2. 问："你还用过哪些类似产品？"
3. 问："你为什么选择我们/竞品？"
4. 列出所有被提及的产品

**示例**：
```
AI产品用户访谈典型发现

Q："你还用过哪些AI对话产品？"
  → 文心一言：高频提及
  → 通义千问：高频提及
  → Kimi：中频提及
  → ChatGPT：部分提及
  → 讯飞星火：部分提及

结论：文心一言、通义千问通常是核心直接竞品
```

#### 方法2：应用商店搜索

**步骤**：
1. 在App Store/安卓应用商店搜索关键词（"AI对话""AI助手""智能助手"）
2. 查看搜索结果前20个App
3. 筛选：功能相似 + 下载量>10万

**工具**：
- **七麦数据**（替代App Annie）：查看App排名、下载量、关键词排名
- **蝉大师**：App Store ASO分析

#### 方法3：融资新闻追踪

**逻辑**：获得融资的AI产品 = 潜在威胁

**步骤**：
1. 订阅36氪、IT桔子、鲸准等融资数据库
2. 设置关键词提醒（"AI对话""大模型""AIGC"）
3. 每周查看新融资项目

**示例**：
```
2026年1月，某AI产品获得A轮融资
  → 产品团队立即研究：产品定位、创始团队、技术路线
  → 判断：该产品主打"企业AI助手"（B2B），与自身产品（B2C）不直接竞争
  → 结论：列入"间接竞品"清单，定期关注
```

#### 方法4：社交媒体监测

**步骤**：
1. 在微博/小红书/知乎搜索"AI对话推荐""最好用的AI"
2. 查看用户讨论最多的产品
3. 爬取评论，统计提及频率

**工具**：
- **新榜**：微信公众号/视频号数据
- **新红**：小红书数据
- **知微数据**：微博数据

## 3. 竞品分析的4个层次

### 3.1 分析框架：从表到里的4层

```
竞品分析的4个层次（由浅入深）

Layer 4: 战略定位 (Strategic Positioning)
  └─ 公司愿景、市场定位、长期战略
     ├─ 分析难度：⭐⭐⭐⭐⭐ (需要推断)
     ├─ 价值：极高（决定长期竞争态势）
     └─ 示例："通义千问的战略是'All in企业市场'"

Layer 3: 商业模式 (Business Model)
  └─ 如何赚钱、定价策略、成本结构
     ├─ 分析难度：⭐⭐⭐⭐ (部分公开)
     ├─ 价值：高（影响自己定价决策）
     └─ 示例："Kimi学术版¥149/月，法律版¥499/月"

Layer 2: 技术架构 (Technical Architecture)
  └─ 使用什么模型、算力规模、技术栈
     ├─ 分析难度：⭐⭐⭐ (可通过逆向工程)
     ├─ 价值：中（了解技术差距）
     └─ 示例："豆包疑似使用字节自研大模型+GPT-4混合"

Layer 1: 产品功能 (Product Features)
  └─ 有哪些功能、界面设计、用户体验
     ├─ 分析难度：⭐ (亲自使用即可)
     ├─ 价值：中（快速对标功能）
     └─ 示例："文心一言有方言支持，豆包没有"
```

### 3.2 Layer 1：产品功能分析（最基础）

**分析内容**：

| 维度 | 分析要点 | 示例 |
|------|---------|------|
| **核心功能** | 有哪些功能？每个功能体验如何？ | 文心一言：对话/绘画/代码/方言 |
| **UI/UX** | 界面设计、交互流程、易用性 | 豆包界面简洁，文心功能入口多 |
| **性能** | 响应速度、准确率、稳定性 | Kimi处理20万字需要30秒 |
| **定价** | 免费/付费功能划分、价格 | 豆包¥99/月，文心一言免费 |
| **平台** | 支持哪些平台（Web/iOS/Android） | 通义千问有飞书插件，豆包没有 |

**工具**：
- **功能对比表**（Excel/飞书表格）
- **用户旅程地图**（Figma/即时设计）

**示例：豆包 vs 文心一言功能对比（2026年Q1）**

| 功能 | 豆包 | 文心一言 | 差异 |
|------|------|---------|------|
| 基础对话 | ✅ 免费 | ✅ 免费 | 持平 |
| 代码生成 | ✅ 专业版¥99 | ✅ 免费 | 文心领先 |
| 图片生成 | ✅ 专业版 | ✅ 免费 | 文心领先 |
| 文件上传 | ✅ 专业版 | ❌ 无 | 豆包领先 |
| 代码解释器 | ✅ 专业版 | ❌ 无 | 豆包领先 |
| 方言支持 | ❌ 无 | ✅ 粤语/四川话 | 文心领先 |
| 多轮对话 | ✅ 100轮 | ✅ 60轮 | 豆包领先 |
| 响应速度 | 2.5秒 | 3.2秒 | 豆包领先 |

**结论**：
- 文心一言"免费策略"激进（代码/绘画免费），豆包需考虑跟进
- 豆包在"专业功能"（文件上传/代码解释器）领先
- 文心的"方言支持"是差异化优势，豆包短期难以复制

---

### 3.3 Layer 2：技术架构分析

**分析内容**：

| 维度 | 分析要点 | 如何获取 |
|------|---------|---------|
| **模型** | 使用什么大模型？参数规模？ | 技术博客、论文、逆向推断 |
| **算力** | GPU/TPU规模、成本 | 招聘JD、行业报告 |
| **技术栈** | 前端/后端框架、数据库 | F12查看网页源码、API监测 |
| **数据** | 训练数据来源、规模 | 官方披露、推测 |

**逆向工程技巧**（合法范围内）：

```
技术架构逆向推断

1. F12查看网页源码
   ├─ 前端框架：React/Vue/Next.js?
   ├─ API endpoint：/v1/chat/completions
   └─ 推断：使用OpenAI兼容接口

2. 抓包分析（Chrome DevTools - Network）
   ├─ 请求参数：model: "gpt-4-turbo"
   ├─ 响应时间：平均2.5秒
   └─ 推断：可能使用GPT-4 API + 缓存优化

3. 招聘JD分析
   ├─ 职位："PyTorch算法工程师"
   ├─ 要求："熟悉Transformer、RLHF"
   └─ 推断：自研大模型，使用PyTorch框架

4. 论文/博客
   ├─ 官方技术博客："我们的模型有1000亿参数"
   └─ 推断：百亿级模型，接近GPT-3.5水平
```

**案例：推断豆包的技术架构（2026年）**

| 证据来源 | 证据 | 推断 |
|---------|------|------|
| **F12源码** | 使用Next.js + React | 现代化前端技术栈 |
| **API监测** | endpoint: `/api/chat`, 返回格式类似OpenAI | 可能基于OpenAI API改造 |
| **招聘JD** | "熟悉LLaMA、Transformer" | 自研模型基于开源改进 |
| **官方博客** | "我们的模型在中文语料上专门优化" | 中文能力强于通用模型 |
| **性能测试** | 响应速度2.5秒，比ChatGPT快 | 国内部署，网络延迟低 |

**结论**：
- 豆包可能使用**字节自研模型** + **GPT-4 API混合**
- 前端技术栈现代化（Next.js）
- 国内部署，速度优势

### 3.4 Layer 3：商业模式分析

**分析内容**：

| 维度 | 分析要点 | 示例 |
|------|---------|------|
| **收入来源** | 订阅/广告/API/企业版？ | Kimi：70%订阅 + 30%企业版 |
| **定价策略** | 免费增值/完全付费/按量计费？ | 豆包：免费版+¥99专业版 |
| **成本结构** | 最大成本是什么？ | AI产品：算力成本占70% |
| **单位经济** | CAC/LTV/毛利率 | 需要推算 |
| **融资情况** | 融资轮次、金额、投资方 | IT桔子查询 |

**定价策略对比（2026年中国AI对话产品）**

| 产品 | 免费版 | 付费版 | 定价策略 | 商业逻辑 |
|------|-------|-------|---------|---------|
| **豆包** | 基础对话 | ¥99/月（高级功能） | 免费增值 | 字节流量优势，免费吸引用户，专业用户付费 |
| **文心一言** | 几乎全功能免费 | ❌ 暂无付费版 | 完全免费 | 百度战略投入，抢占市场份额，暂不考虑变现 |
| **Kimi** | 基础对话 | ¥149-¥999/月（垂直版本） | 垂直分层定价 | 专业用户愿意为"超长文本"付费，高ARPU |
| **通义千问** | 基础对话 | 企业定制（¥50k+/年） | B2B为主 | 阿里云生态，企业客户为主要营收 |

**关键洞察**：

```
AI产品免费策略的竞争压力（2026年）

问题：文心一言全功能免费策略 → 付费产品面临压力

应对策略（3选1）：
  ① 降价：降低付费门槛（风险：收入减少）
  ② 跟进免费：核心功能免费（风险：短期亏损）
  ③ 差异化：强化专业功能（风险：需要开发周期）

行业趋势：
  → 头部产品选择差异化策略
  → 推出专业工作站功能（文件上传+代码解释+数据分析）
  → 守住高价值付费用户群体
```

### 3.5 Layer 4：战略定位分析（最难但最重要）

**分析内容**：

| 维度 | 分析要点 | 如何推断 |

## 6. 竞品分析实战案例

### 6.1 案例：AI产品如何分析垂直竞品？（以Kimi为例）

**背景**（2025年）：
- Kimi月活[曾达3600万峰值](https://m.thepaper.cn/newsDetail_forward_32203522)，后因竞争加剧有所下降
- 行业观察：专业用户（程序员、学术）向垂直工具迁移
- 任务：理解垂直竞品策略，制定应对方案

**Step 1：产品功能分析（Layer 1）**

```
亲自体验Kimi（产品对比分析）

测试场景1：上传长篇PDF文档
  ├─ Kimi：快速处理，准确总结研究问题/方法/结论
  ├─ 通用AI产品：文件处理能力有限
  └─ 差距：Kimi在长文本处理上建立技术优势

测试场景2：处理超长文本（10万字+）
  ├─ Kimi：完整理解，能回答全文细节
  ├─ 通用AI产品：上下文窗口限制，部分内容"遗忘"
  └─ 差距：Kimi技术壁垒（超长上下文）

测试场景3：日常对话
  ├─ Kimi：回复偏学术化、严谨
  ├─ 通用AI产品：回复更口语化、轻松
  └─ 差距：定位不同（Kimi偏专业，通用产品偏大众）

结论：
  → Kimi核心优势 = "超长文本处理"
  → 通用产品短期难以复制（技术投入大）
```

**Step 2：用户画像分析**

```
用户访谈典型反馈（垂直AI工具用户）

用户A（博士生）：
  "我每周要读多篇长论文，Kimi能一次处理完整篇，
   非常方便。"

用户B（律师）：
  "Kimi能分析长篇合同，高亮风险点，并标注具体位置，
   这个功能非常实用。"

用户C（程序员）：
  "我用Kimi看长文档（API文档、技术白皮书），
   省了很多时间。"

共性：
  → 垂直工具用户多为"专业用户"（学术/法律/技术）
  → 核心需求："处理超长文本"（50页+）
  → 付费意愿：较高（专业场景价值明确）
```

**Step 3：商业模式分析（Layer 3）**

```
Kimi的商业模式特点

定价策略：
  ├─ 分层定价（基础版/专业版）
  ├─ 垂直场景溢价（学术/法律/企业）
  └─ 按使用量计费选项

商业模式特点：
  ├─ 团队规模：[100-200人](https://zhuanlan.zhihu.com/p/688302538)（相对精简）
  ├─ 融资情况：[5亿美元多轮融资](https://finance.sina.com.cn/tech/roll/2026-01-08/doc-inhfqyhr1234958.shtml)
  ├─ 市场定位：垂直专业场景
  └─ 用户特征：专业用户付费意愿高

成本结构特点：
  ├─ 算力成本：处理超长文本，单用户成本较高
  ├─ 人力成本：技术团队为主
  └─ 营销成本：垂直渠道精准投放

关键洞察：
  → Kimi以精简团队服务垂直市场
  → 专业用户付费率显著高于通用产品
  → 垂直深耕策略成效明显
```

**Step 4：战略定位分析（Layer 4）**

```
Kimi的长期战略推断

证据1：团队招聘
  ├─ 大量招聘"垂直行业PM"（法律/医疗/金融）
  └─ 推断：深耕垂直场景

证据2：CEO访谈（2026年1月）
  "我们不追求月活第一，我们要做专业用户最信赖的AI工具。"
  └─ 推断：定位专业用户，不做大众市场

  ├─ 上线"可溯源引用"（学术用户需求）
  ├─ 上线"批量PDF处理"（法律用户需求）
  └─ 推断：围绕专业场景深挖

```

## 8. 常见错误与陷阱

### 8.1 错误1：只分析直接竞品，忽略间接竞品

**案例**：

```
行业观察：AI产品只关注直接竞品

问题：
  → 专业用户（程序员等）持续流失
  → 调研发现：流失到IDE集成工具（间接竞品）
  → 原因：未监测"满足同样需求"的间接竞品

教训：
  ✅ 除了直接竞品，必须监测间接竞品
  ✅ 定期问用户："你还用什么产品替代我们？"
  ✅ 关注不同产品形态的替代方案
```

### 8.2 错误2：只看公开信息，不亲自使用

**案例**：

```
行业教训：只依赖公开信息，不亲自使用

问题：
  → 错过了竞品的"隐藏功能"（只有付费用户能用）
  → 对竞品用户体验的理解停留在表面
  → 无法发现真实使用中的优缺点

教训：
  ✅ 必须注册账号、付费试用、深度体验
  ✅ 模拟真实用户场景（工作/学习/娱乐）
  ✅ 定期体验竞品（建议每月至少3天）
```

### 8.3 错误3：分析完没有行动

**案例**：

```
某AI产品团队：
  ├─ 每月花10小时做竞品分析
  ├─ 生成20页竞品报告
  └─ 然后...放在文档里，再也没人看

结果：
  → 竞品分析变成"摆设"
  → 产品决策还是靠"拍脑袋"

教训：
  ✅ 竞品分析必须输出"行动计划"
  ✅ 每次分析后，回答3个问题：
      1. 我们要学习竞品的什么？（借鉴）
      2. 我们要避免竞品的什么？（避坑）
      3. 我们要差异化在哪里？（定位）
  ✅ 制定具体行动（2周内实施）
```

### 8.4 错误4：过度模仿竞品

**案例**：

```
行业教训：盲目模仿竞品免费策略

决策：看到头部产品免费策略，立即跟进

问题：
  → 营收大幅下降
  → 资金压力剧增
  → 未考虑资源禀赋差异

教训：
  ✅ 竞品的成功策略，不一定适合自己
  ✅ 头部产品有生态补贴能力，中小产品需谨慎
  ✅ 盲目模仿可能导致商业模式崩溃
  ✅ 正确做法：分析竞品策略背后的"资源禀赋"，判断自己是否具备
```

**答案**：

**3个策略选项**：

| 策略 | 优点 | 缺点 | 风险 |
|------|------|------|------|

   - 后果：监管处罚、刑事责任
