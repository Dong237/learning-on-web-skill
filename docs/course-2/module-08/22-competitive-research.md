# 22 | 研究竞争对手 Researching Competitors

## TL;DR (3分钟速读)

竞品分析是通过系统性研究竞争对手的产品、策略、优劣势，来指导自己产品决策的过程。

**核心要点**：
- **定义竞争对手**：直接竞争对手（同功能同用户）、间接竞争对手（替代解决方案）、潜在竞争对手（可能进入的玩家）
- **4个层次分析**：产品功能 → 商业模式 → 技术架构 → 战略定位
- **信息来源**：公开渠道（官网/App/社交媒体）、第三方数据（艾瑞/易观/Sensor Tower）、一线情报（用户访谈/行业峰会）、逆向工程（产品拆解）
- **2026中国AI产品竞争格局**：大模型赛道"百模大战"（通义/文心/豆包/Kimi/智谱等20+玩家）、AIGC工具赛道（文生图/文生视频/AI设计）、垂直场景AI（法律/医疗/教育）

**关键工具**：
```
信息收集工具（中国版）：
- 公司信息：天眼查、企查查
- 流量数据：七麦数据（App）、5118（Web）、艾瑞咨询
- 用户评价：应用商店评论、小红书、知乎、微博
- 技术架构：产品拆解、API监测
- 财务数据：IT桔子、36氪、企名片
```

**实战建议**：
1. **定期更新**：AI产品迭代快，每月更新竞品信息（不是季度）
2. **聚焦关键竞品**：深度分析3-5个核心竞品，而非泛泛研究20个
3. **用户视角**：亲自使用竞品（注册账号、付费体验），而非只看报告
4. **合法合规**：绝不窃取商业机密、源代码、用户数据

## 1. 什么是竞品分析

### 1.1 定义

**竞品分析 (Competitive Analysis)** = 系统性研究竞争对手的**产品、策略、优劣势**，以指导自己产品决策的过程。

**核心目标**：
1. **知己知彼**：了解竞争对手在做什么、做得如何
2. **寻找机会**：发现竞品的弱点，自己的差异化空间
3. **学习借鉴**：吸收竞品的优秀实践
4. **预判趋势**：通过竞品动态，预测行业走向

### 1.2 为什么竞品分析对AI产品特别重要？

```
AI产品 vs 传统软件：竞品分析的差异

传统软件（如Excel）：
  ├─ 竞争格局：稳定（Google Sheets、WPS表格）
  ├─ 迭代周期：年度大版本更新
  ├─ 技术门槛：高（10年开发经验）
  └─ 竞品分析频率：每季度1次

AI产品（如AI对话）：
  ├─ 竞争格局：极不稳定（每月新玩家、每周新功能）
  ├─ 迭代周期：周更新、日更新
  ├─ 技术门槛：相对低（开源模型+API调用）
  └─ 竞品分析频率：每月1次甚至每周1次 ⚠️

AI产品的特殊挑战：
1. 新玩家频繁出现（2024-2026年中国大模型从5个→20+个）
2. 功能快速同质化（竞品上线新功能，1周内被抄袭）
3. 技术突破难预测（GPT-5发布可能颠覆整个格局）
4. 用户迁移成本低（切换AI产品只需10分钟）
```

**2026真实数据**：
- 某AI对话产品PM：**每周**花5小时做竞品分析（vs 传统软件PM每季度5小时）
- 豆包产品团队：设置"竞品雷达"，**每天**自动抓取竞品更新（新功能、定价变化、用户评价）

## 2. 识别竞争对手：3类竞争对手

### 2.1 竞争对手的3个层次

```
竞争对手的3圈层模型

           ┌──────────────────────────────┐
           │  潜在竞争对手 (Potential)      │
           │  可能进入市场的玩家            │
           │  ┌────────────────────────┐  │
           │  │ 间接竞争对手 (Indirect) │  │
           │  │ 替代解决方案            │  │
           │  │ ┌──────────────────┐  │  │
           │  │ │ 直接竞争对手      │  │  │
           │  │ │ (Direct)         │  │  │
           │  │ │ 同功能同用户      │  │  │
           │  │ └──────────────────┘  │  │
           │  └────────────────────────┘  │
           └──────────────────────────────┘
```

#### 层次1：直接竞争对手 (Direct Competitors)

**定义**：与你的产品**功能相同、目标用户相同**的竞品

**识别标准**：
- ✅ 功能重叠度>70%
- ✅ 目标用户群体相同
- ✅ 商业模式相似
- ✅ 用户在两者之间做选择

**示例：豆包的直接竞争对手（2026年）**

| 竞品 | 功能重叠度 | 目标用户 | 为什么是直接竞争 |
|------|----------|---------|----------------|
| **文心一言** | 90% | 通用AI对话用户 | 百度出品，功能几乎一致，用户在选择"用豆包还是文心" |
| **通义千问** | 85% | 通用AI对话用户 | 阿里出品，主打工作提效，与豆包重叠 |
| **Kimi** | 60% | 专业用户（超长文本） | 虽然定位垂直，但核心功能仍是AI对话，有重叠 |
| **讯飞星火** | 80% | 通用AI对话用户 | 科大讯飞出品，语音优势，但整体功能相似 |

**优先级**：🔥🔥🔥 **最高**（必须深度分析）

---

#### 层次2：间接竞争对手 (Indirect Competitors)

**定义**：功能不同，但**解决同样用户需求**的替代方案

**识别标准**：
- ✅ 解决同样的"Job to Be Done"
- ✅ 用户场景重叠
- ❌ 但产品形态不同

**示例：豆包的间接竞争对手**

| 竞品 | 用户需求 | 为什么是间接竞争 | 威胁程度 |
|------|---------|----------------|---------|
| **Notion AI** | 工作文档撰写 | 用户本来想用豆包写文档，但Notion AI集成在编辑器里更方便 | 中 |
| **飞书智能伙伴** | 企业办公提效 | 企业用户用飞书智能伙伴处理工作任务，就不用豆包了 | 高 |
| **GitHub Copilot** | 代码编写 | 程序员用Copilot写代码，就不用豆包了 | 中 |
| **ChatGPT** | 通用AI对话 | 虽然在中国需要翻墙，但部分高端用户仍使用（English能力强） | 高 |

**关键洞察**：间接竞争对手往往被忽视，但可能**抢走你最有价值的用户**

**案例**：
```
2025年Q3，豆包发现：
  - 程序员用户流失率25%（最高价值用户群）
  - 调研发现：80%流失到GitHub Copilot（间接竞品）
  - 原因：Copilot集成在IDE里，用豆包需要切换窗口，太麻烦

应对策略：
  ✅ 2025年Q4，豆包推出VS Code插件
  ✅ 程序员流失率降至12%
```

**优先级**：🔥🔥 **高**（需要关注，尤其是抢走高价值用户的竞品）

---

#### 层次3：潜在竞争对手 (Potential Competitors)

**定义**：现在不是竞品，但**未来可能进入市场**的玩家

**识别标准**：
- 有技术能力
- 有用户基础
- 有资金实力
- 有进入动机

**示例：豆包的潜在竞争对手（2026年预判）**

| 潜在竞品 | 当前业务 | 为什么可能进入 | 进入概率 |
|---------|---------|--------------|---------|
| **微信** | 社交 | 10亿用户，可以在微信内置AI对话（类似微信AI） | 80% |
| **滴滴** | 出行 | 有AI技术积累，可能推出"滴滴AI助手"（车上闲聊场景） | 30% |
| **拼多多** | 电商 | 可能推出"AI购物助手"（帮用户找商品、比价） | 50% |
| **美团** | 本地生活 | 可能推出"AI生活助手"（订餐、订酒店、规划行程） | 60% |

**关键动作**：
1. **监测信号**：关注潜在竞品的招聘（如"美团招聘大模型算法工程师"）
2. **提前布局**：在潜在竞品进入前，抢占核心场景（如豆包提前做"购物助手"功能）

**优先级**：🔥 **中**（长期关注，但不必立即深度分析）

### 2.2 如何识别竞争对手？4个方法

#### 方法1：用户视角（最准确）

**问题**："如果用户不用你的产品，他会用什么？"

**步骤**：
1. 访谈10-20个真实用户/潜在用户
2. 问："你还用过哪些类似产品？"
3. 问："你为什么选择我们/竞品？"
4. 列出所有被提及的产品

**示例**：
```
豆包用户访谈（n=30）

Q："你还用过哪些AI对话产品？"
  → 文心一言：23次提及
  → 通义千问：18次提及
  → Kimi：12次提及
  → ChatGPT：8次提及
  → 讯飞星火：5次提及

结论：文心一言、通义千问是最核心的直接竞品
```

#### 方法2：应用商店搜索

**步骤**：
1. 在App Store/安卓应用商店搜索关键词（"AI对话""AI助手""智能助手"）
2. 查看搜索结果前20个App
3. 筛选：功能相似 + 下载量>10万

**工具**：
- **七麦数据**（替代App Annie）：查看App排名、下载量、关键词排名
- **蝉大师**：App Store ASO分析

#### 方法3：融资新闻追踪

**逻辑**：获得融资的AI产品 = 潜在威胁

**步骤**：
1. 订阅36氪、IT桔子、鲸准等融资数据库
2. 设置关键词提醒（"AI对话""大模型""AIGC"）
3. 每周查看新融资项目

**示例**：
```
2026年1月，某AI产品获得A轮¥5000万融资
  → 豆包PM立即研究：产品定位、创始团队、技术路线
  → 判断：该产品主打"企业AI助手"（B2B），与豆包（B2C）不直接竞争
  → 结论：列入"间接竞品"清单，每季度关注
```

#### 方法4：社交媒体监测

**步骤**：
1. 在微博/小红书/知乎搜索"AI对话推荐""最好用的AI"
2. 查看用户讨论最多的产品
3. 爬取评论，统计提及频率

**工具**：
- **新榜**：微信公众号/视频号数据
- **新红**：小红书数据
- **知微数据**：微博数据

## 3. 竞品分析的4个层次

### 3.1 分析框架：从表到里的4层

```
竞品分析的4个层次（由浅入深）

Layer 4: 战略定位 (Strategic Positioning)
  └─ 公司愿景、市场定位、长期战略
     ├─ 分析难度：⭐⭐⭐⭐⭐ (需要推断)
     ├─ 价值：极高（决定长期竞争态势）
     └─ 示例："通义千问的战略是'All in企业市场'"

Layer 3: 商业模式 (Business Model)
  └─ 如何赚钱、定价策略、成本结构
     ├─ 分析难度：⭐⭐⭐⭐ (部分公开)
     ├─ 价值：高（影响自己定价决策）
     └─ 示例："Kimi学术版¥149/月，法律版¥499/月"

Layer 2: 技术架构 (Technical Architecture)
  └─ 使用什么模型、算力规模、技术栈
     ├─ 分析难度：⭐⭐⭐ (可通过逆向工程)
     ├─ 价值：中（了解技术差距）
     └─ 示例："豆包疑似使用字节自研大模型+GPT-4混合"

Layer 1: 产品功能 (Product Features)
  └─ 有哪些功能、界面设计、用户体验
     ├─ 分析难度：⭐ (亲自使用即可)
     ├─ 价值：中（快速对标功能）
     └─ 示例："文心一言有方言支持，豆包没有"
```

### 3.2 Layer 1：产品功能分析（最基础）

**分析内容**：

| 维度 | 分析要点 | 示例 |
|------|---------|------|
| **核心功能** | 有哪些功能？每个功能体验如何？ | 文心一言：对话/绘画/代码/方言 |
| **UI/UX** | 界面设计、交互流程、易用性 | 豆包界面简洁，文心功能入口多 |
| **性能** | 响应速度、准确率、稳定性 | Kimi处理20万字需要30秒 |
| **定价** | 免费/付费功能划分、价格 | 豆包¥99/月，文心一言免费 |
| **平台** | 支持哪些平台（Web/iOS/Android） | 通义千问有飞书插件，豆包没有 |

**工具**：
- **功能对比表**（Excel/飞书表格）
- **用户旅程地图**（Figma/即时设计）

**示例：豆包 vs 文心一言功能对比（2026年Q1）**

| 功能 | 豆包 | 文心一言 | 差异 |
|------|------|---------|------|
| 基础对话 | ✅ 免费 | ✅ 免费 | 持平 |
| 代码生成 | ✅ 专业版¥99 | ✅ 免费 | 文心领先 |
| 图片生成 | ✅ 专业版 | ✅ 免费 | 文心领先 |
| 文件上传 | ✅ 专业版 | ❌ 无 | 豆包领先 |
| 代码解释器 | ✅ 专业版 | ❌ 无 | 豆包领先 |
| 方言支持 | ❌ 无 | ✅ 粤语/四川话 | 文心领先 |
| 多轮对话 | ✅ 100轮 | ✅ 60轮 | 豆包领先 |
| 响应速度 | 2.5秒 | 3.2秒 | 豆包领先 |

**结论**：
- 文心一言"免费策略"激进（代码/绘画免费），豆包需考虑跟进
- 豆包在"专业功能"（文件上传/代码解释器）领先
- 文心的"方言支持"是差异化优势，豆包短期难以复制

---

### 3.3 Layer 2：技术架构分析

**分析内容**：

| 维度 | 分析要点 | 如何获取 |
|------|---------|---------|
| **模型** | 使用什么大模型？参数规模？ | 技术博客、论文、逆向推断 |
| **算力** | GPU/TPU规模、成本 | 招聘JD、行业报告 |
| **技术栈** | 前端/后端框架、数据库 | F12查看网页源码、API监测 |
| **数据** | 训练数据来源、规模 | 官方披露、推测 |

**逆向工程技巧**（合法范围内）：

```
技术架构逆向推断

1. F12查看网页源码
   ├─ 前端框架：React/Vue/Next.js?
   ├─ API endpoint：/v1/chat/completions
   └─ 推断：使用OpenAI兼容接口

2. 抓包分析（Chrome DevTools - Network）
   ├─ 请求参数：model: "gpt-4-turbo"
   ├─ 响应时间：平均2.5秒
   └─ 推断：可能使用GPT-4 API + 缓存优化

3. 招聘JD分析
   ├─ 职位："PyTorch算法工程师"
   ├─ 要求："熟悉Transformer、RLHF"
   └─ 推断：自研大模型，使用PyTorch框架

4. 论文/博客
   ├─ 官方技术博客："我们的模型有1000亿参数"
   └─ 推断：百亿级模型，接近GPT-3.5水平
```

**案例：推断豆包的技术架构（2026年）**

| 证据来源 | 证据 | 推断 |
|---------|------|------|
| **F12源码** | 使用Next.js + React | 现代化前端技术栈 |
| **API监测** | endpoint: `/api/chat`, 返回格式类似OpenAI | 可能基于OpenAI API改造 |
| **招聘JD** | "熟悉LLaMA、Transformer" | 自研模型基于开源改进 |
| **官方博客** | "我们的模型在中文语料上专门优化" | 中文能力强于通用模型 |
| **性能测试** | 响应速度2.5秒，比ChatGPT快 | 国内部署，网络延迟低 |

**结论**：
- 豆包可能使用**字节自研模型** + **GPT-4 API混合**
- 前端技术栈现代化（Next.js）
- 国内部署，速度优势

### 3.4 Layer 3：商业模式分析

**分析内容**：

| 维度 | 分析要点 | 示例 |
|------|---------|------|
| **收入来源** | 订阅/广告/API/企业版？ | Kimi：70%订阅 + 30%企业版 |
| **定价策略** | 免费增值/完全付费/按量计费？ | 豆包：免费版+¥99专业版 |
| **成本结构** | 最大成本是什么？ | AI产品：算力成本占70% |
| **单位经济** | CAC/LTV/毛利率 | 需要推算 |
| **融资情况** | 融资轮次、金额、投资方 | IT桔子查询 |

**定价策略对比（2026年中国AI对话产品）**

| 产品 | 免费版 | 付费版 | 定价策略 | 商业逻辑 |
|------|-------|-------|---------|---------|
| **豆包** | 基础对话 | ¥99/月（高级功能） | 免费增值 | 字节流量优势，免费吸引用户，专业用户付费 |
| **文心一言** | 几乎全功能免费 | ❌ 暂无付费版 | 完全免费 | 百度战略投入，抢占市场份额，暂不考虑变现 |
| **Kimi** | 基础对话 | ¥149-¥999/月（垂直版本） | 垂直分层定价 | 专业用户愿意为"超长文本"付费，高ARPU |
| **通义千问** | 基础对话 | 企业定制（¥50k+/年） | B2B为主 | 阿里云生态，企业客户为主要营收 |

**关键洞察**：

```
豆包的定价困境（2026年Q1）

问题：文心一言免费策略 → 豆包付费用户流失

数据：
  ├─ 2025年Q4：豆包付费用户100万，付费率11%
  ├─ 2026年Q1：文心全功能免费后，豆包付费用户降至75万

  ① 降价：¥99 → ¥49（风险：收入减半）

  → 3个月后推出"AI工作站"（文件上传+代码解释+数据分析）
  → 付费用户恢复至95万，付费率13%

- 任务：深度分析Kimi，找出应对策略

**Step 1：产品功能分析（Layer 1）**

```

  ├─ Kimi：30秒处理完成，准确总结研究问题/方法/结论
  ├─ 豆包：❌ 不支持PDF上传

  ├─ Kimi：完整理解，能回答结尾剧情
  ├─ 豆包：只能处理前1万字，后面"遗忘"
  └─ 差距：Kimi技术壁垒（超长上下文）

测试场景3：日常对话
  ├─ Kimi：回复偏学术化、严谨
  ├─ 豆包：回复更口语化、轻松
  └─ 差距：定位不同（Kimi偏专业，豆包偏大众）

结论：
  → Kimi核心优势 = "超长文本处理"
  → 豆包短期无法复制（技术门槛高）

**Step 2：用户画像分析**

  ✅ 产品迭代快（周更新）
  ✅ 年轻用户基础（18-35岁）

Weaknesses（豆包劣势）：
  ❌ 不支持超长文本（技术短板）
  ❌ 专业用户流失（学术/法律用户）
  ❌ 付费率低于Kimi（11% vs 18%）

Opportunities（机会）：
  ✅ Kimi专注垂直，豆包可以守住大众市场
  ✅ 开发"中等长度文本"功能（10-20页，vs Kimi的50+页）
  ✅ 与Kimi形成互补（而非正面竞争）

Threats（威胁）：
  ❌ 高价值用户持续流失到Kimi
  ❌ Kimi可能从垂直扩展到大众市场

决策：
  ✅ 豆包不跟进"超长文本"（技术投入大，ROI低）
  ✅ 聚焦"大众用户"（娱乐/轻办公场景）
  ✅ 开发"文件上传+代码解释器"（满足轻度专业需求）
  ✅ 加强年轻用户运营（游戏/二次元场景）

结果（2026年Q2）：
  → 豆包与Kimi形成"互补共存"
  → 豆包守住大众市场（月活8500万 → 1亿）
  → Kimi深耕专业市场（月活2000万 → 3000万）
  → 双方用户重叠度降至15%（vs 之前40%）

## 8. 常见错误与陷阱

### 8.1 错误1：只分析直接竞品，忽略间接竞品

**案例**：

```
2025年Q2，豆包只盯着文心一言、通义千问（直接竞品）

结果：
  → 程序员用户流失25%
  → 调研发现：流失到GitHub Copilot（间接竞品）
  → 原因：豆包没有监测"代码AI"这个间接竞品

教训：
  ✅ 除了直接竞品，必须监测"满足同样需求"的间接竞品
  ✅ 定期问用户："你还用什么产品替代我们？"
```

### 8.2 错误2：只看公开信息，不亲自使用

**案例**：

```
某AI产品PM只看竞品官网、报告，从不注册使用

结果：
  → 错过了竞品的"隐藏功能"（只有付费用户能用）
  → 对竞品用户体验的理解停留在表面

教训：
  ✅ 必须注册账号、付费试用、深度体验
  ✅ 模拟真实用户场景（工作/学习/娱乐）
  ✅ 每月至少体验3天竞品
```

### 8.3 错误3：分析完没有行动

**案例**：

```
某AI产品团队：
  ├─ 每月花10小时做竞品分析
  ├─ 生成20页竞品报告
  └─ 然后...放在文档里，再也没人看

结果：
  → 竞品分析变成"摆设"
  → 产品决策还是靠"拍脑袋"

教训：
  ✅ 竞品分析必须输出"行动计划"
  ✅ 每次分析后，回答3个问题：
      1. 我们要学习竞品的什么？（借鉴）
      2. 我们要避免竞品的什么？（避坑）
      3. 我们要差异化在哪里？（定位）
  ✅ 制定具体行动（2周内实施）
```

### 8.4 错误4：过度模仿竞品

**案例**：

```
2025年，某AI产品看到文心一言免费策略成功

决策：立即跟进，全功能免费

结果：
  → 月营收从¥5000万降至¥500万（-90%）
  → 6个月后资金链断裂

教训：
  ✅ 竞品的成功策略，不一定适合你
  ✅ 文心有百度搜索补贴，你没有
  ✅ 盲目模仿 = 自杀
  ✅ 正确做法：分析竞品策略背后的"资源禀赋"，判断自己是否具备
```

   - 后果：刑事责任、巨额赔偿

   - 后果：监管处罚、刑事责任

**判断标准**：
- ✅ **合法**：使用公开信息 + 正常使用产品（自己的账号）
- ❌ **非法**：窃取商业机密 + 侵犯用户隐私 + 破坏系统安全
