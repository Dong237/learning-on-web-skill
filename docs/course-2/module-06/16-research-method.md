## 市场研究方法全景图

市场研究不是单一技能，而是**方法工具箱**。PM需要根据研究目标、资源、时间选择合适的工具组合。

### 研究方法四象限

```
           探索性 Exploratory
                 ↑
                 │
     ┌───────────┼───────────┐
     │  定性探索  │  定量探索  │
     │Qualitative│Quantitative│
     │ Explor.   │ Explor.    │
     ├───────────┼───────────┤
定性  │ ● 用户访谈 │ ● 问卷调查 │ 定量
Qual. │ ● 焦点小组 │ ● A/B测试  │ Quant.
     │ ● 日记研究 │ ● 行为分析 │
     │ ● 民族志   │ ● 大数据挖掘│
     ├───────────┼───────────┤
     │  定性验证  │  定量验证  │
     │Qualitative│Quantitative│
     │ Valid.    │ Valid.     │
     └───────────┼───────────┘
                 │
                 ↓
           验证性 Confirmatory
```

### 定性 vs 定量：核心差异

| 维度 | 定性研究 Qualitative | 定量研究 Quantitative |
|------|---------------------|----------------------|
| **核心问题** | Why? How? (为什么/怎么样) | What? How many? (是什么/多少) |
| **样本量** | 小(5-30人) | 大(100-10000+人) |
| **数据类型** | 文本、视频、观察笔记 | 数字、统计指标 |
| **分析方法** | 主题编码、模式识别 | 统计检验、回归分析 |
| **结果** | 深度洞察、假设生成 | 可推广结论、验证假设 |
| **适用阶段** | 早期探索、问题发现 | 后期验证、规模评估 |
| **成本** | 时间成本高、执行成本低 | 时间成本低、执行成本高 |
| **代表工具** | 用户访谈、焦点小组 | 问卷调查、A/B测试 |

> **💡 实战洞察：混合方法论才是王道**
> 2026年字节跳动豆包团队的多模态研究案例：先用**15场用户访谈**发现"用户希望AI既能聊天又能画图还能写代码"的混合需求（定性），再用**50万用户的A/B测试**验证"多模态入口放在哪里转化率最高"（定量）。纯定性容易陷入"样本偏差"，纯定量容易陷入"不知道为什么"。**混合才能又深又广**。

## 定性研究：探索未知的利器

定性研究是**打开黑盒子**的钥匙——当你不知道用户真正想要什么、痛点在哪里、如何描述问题时，定性研究最有效。

### 常用定性方法

#### 1. 深度访谈 In-Depth Interview

**定义**：一对一半结构化对话，深挖用户行为动机、情感、心智模型。

**适用场景**：
- B2B产品（样本量小、决策复杂）
- 敏感话题（隐私、财务、健康）
- 探索新领域（AI产品早期）

**2026中国AI工具**：
- **飞书妙记** (取代Otter.ai)：AI自动转录、生成访谈摘要
- **腾讯会议云录制** (取代Zoom)：高清录屏 + 智能字幕
- **Notion AI** (中国版)：访谈笔记自动整理、主题提取

**关键技巧**：
- 5W1H追问法：Why背后的Why (连问5次为什么)
- 沉默的力量：用户说完后等3秒，往往有惊喜
- 避免引导性问题："你喜欢这个功能吗？"❌ → "你上次使用这个功能是什么时候？"✅

#### 2. 焦点小组 Focus Group

**定义**：6-10人的小组讨论，观察用户间互动、共识形成过程。

**适用场景**：
- 品牌认知研究（多人讨论更真实）
- 创意测试（观察用户反应）
- 市场细分验证（观察不同群体差异）

**2026中国AI实战**：
月之暗面Kimi团队2026年3月用焦点小组测试"超长文本"价值主张：
- 邀请8位知识工作者（律师、研究员、编剧）
- 现场演示"上传200页PDF，5秒生成摘要"
- **关键发现**：用户最兴奋的不是"速度快"，而是"终于不用自己读了"（情感价值>功能价值）

**常见陷阱**：
- 群体思维（一个人主导讨论，其他人跟风）
- 社会期望偏差（用户说"应该"的答案，而非真实想法）

#### 3. 用户日记研究 Diary Study

**定义**：让用户在真实场景中记录使用体验（文字/照片/视频），持续数天到数周。

**适用场景**：
- 长周期行为研究（健康、教育、理财）
- 低频高价值行为（买房、求职）
- 情境依赖行为（通勤、出差）

**2026中国工具**：
- **问卷星** (取代SurveyMonkey)：定时推送日记问卷
- **微信小程序** + 企业微信：嵌入用户日常流中
- **飞书多维表格**：结构化日记数据收集

#### 4. 民族志研究 Ethnographic Research

**定义**：研究员进入用户真实环境，长时间沉浸式观察（如跟着用户上班一天）。

**适用场景**：
- 复杂工作流研究（企业软件、医疗系统）
- 文化适配研究（国际化产品）
- 创新灵感发现（观察用户"变通行为"）

**2026中国AI案例**：
阿里通义千问企业版团队派PM到杭州某跨境电商公司"蹲点"3天，发现：
- 运营每天要在5个系统间来回切换（痛点！）
- 用Excel手工整理数据后才能用ChatGPT（低效！）
- **洞察**：企业用户需要"AI + 现有系统集成"，而非独立AI工具

## 定量研究：验证假设的标尺

定量研究是**测量和证明**的工具——当你有明确假设、需要验证规模、需要说服老板时，定量研究最有力。

### 常用定量方法

#### 1. 问卷调查 Survey

**定义**：结构化问题收集大量用户反馈，产生统计可靠的数据。

**适用场景**：
- 市场规模估算（TAM/SAM/SOM）
- 功能优先级排序（Kano模型）
- 用户满意度追踪（NPS/CSAT）

**2026中国工具生态**：

| 国外工具 | 中国替代 | 特色功能 |
|---------|---------|---------|
| SurveyMonkey | **问卷星** | AI自动生成题目、智能逻辑跳转 |
| Qualtrics | **腾讯问卷** | 微信生态集成、红包激励 |
| Typeform | **金数据** | 精美UI、支付集成 |

**问卷设计黄金法则**：
- **单一性**：一题只问一件事（"产品好用且便宜吗？"❌）
- **避免双重否定**："你不认为AI不应该收费吗？"❌
- **随机化选项顺序**：避免"总是选第一个"偏差
- **奇数量表**：1-5分 or 1-7分（有中立选项）vs 1-4分（强迫选择）

#### 2. A/B测试 A/B Testing

**定义**：随机分配用户到不同版本，对比关键指标（转化率、留存率等）。

**适用场景**：
- 产品优化决策（按钮颜色、文案、布局）
- 定价策略测试
- 新功能上线前验证

**2026中国AI实战**：
字节豆包2026年5月测试"对话界面"vs"搜索界面"：
- **对照组A**：传统对话框（ChatGPT风格）
- **实验组B**：搜索框 + 智能推荐（Google风格）
- **样本**：100万新用户，随机50/50分配
- **结果**：B组7日留存率高18%，但单次对话轮数少22%
- **决策**：首页用B（拉新），深度功能用A（促活）

**统计显著性计算**：

```
显著性水平 α = 0.05 (常用标准)
样本量 n ≥ (Zα/2)² × p(1-p) / E²

例：估算"豆包付费转化率"（预期5%，误差±1%）
n ≥ (1.96)² × 0.05(0.95) / 0.01² = 1824 用户
```

#### 3. 行为数据分析 Behavioral Analytics

**定义**：分析用户在产品中的真实行为数据（点击流、使用时长、功能路径等）。

**适用场景**：
- 用户旅程分析（用户从哪来、去哪、卡在哪）
- 流失原因诊断
- 功能使用率评估

**2026中国工具栈**：

| 层级 | 国外工具 | 中国替代 | 用途 |
|------|---------|---------|------|
| 埋点采集 | Segment | **神策数据** | 全端埋点SDK |
| 行为分析 | Mixpanel | **诸葛IO** | 漏斗分析、留存分析 |
| 会话回放 | Hotjar | **GrowingIO** | 用户行为录屏 |
| 大数据平台 | Amplitude | **友盟+** | 亿级用户分析 |

**行为分析经典指标**：

```
DAU/MAU比值（粘性）：
- 社交产品：>50%（微信约60%）
- 工具产品：20-40%（豆包约35%）
- 内容产品：10-30%（得到APP约25%）

留存率曲线（Retention Curve）：
Day 1: 40% (首日留存)
Day 7: 25% (周留存)
Day 30: 15% (月留存)
↑ AI产品典型曲线（2026中国数据）
```

#### 4. 大数据挖掘 Big Data Mining

**定义**：从海量数据中用算法发现模式、预测趋势（需数据科学团队支持）。

**适用场景**：
- 用户细分（聚类算法找到隐藏用户群）
- 流失预测（机器学习预测高危用户）
- 推荐系统优化

**2026中国AI案例**：
腾讯元宝用NLP分析200万条用户反馈，发现：
- **高频痛点词云**："不准"(18%)、"太慢"(12%)、"听不懂"(9%)
- **隐藏需求**：15%用户提到"方言支持"（团队之前未关注）
- **行动**：优先开发粤语/四川话模型

## AI产品研究的特殊挑战

AI产品不同于传统软件，研究方法需要适配3大特殊性：

### 挑战1：用户不知道AI能做什么

**传统产品**：用户有明确需求（"我要修图软件"）
**AI产品**：用户不知道边界（"AI能帮我做饭吗？"）

**应对策略**：
- **Show, Don't Tell**：用demo演示替代问卷（"你会用AI写周报吗？"→ 现场演示AI写周报）
- **Wizard of Oz测试**：人工模拟AI响应，测试用户反应（避免过早投入研发）
- **2026案例**：Kimi团队用"人工客服假装AI"测试"超长对话"需求，发现用户期望"跨天记忆"（技术难点）

### 挑战2：AI行为不可预测

**传统产品**：A输入→B输出（确定性）
**AI产品**：A输入→B/C/D输出（随机性）

**应对策略**：
- **边界测试**：故意输入极端case，观察AI崩溃场景
- **幻觉监测**：记录AI"一本正经胡说八道"的频率
- **2026案例**：文心一言团队建立"幻觉数据库"，收集10万+错误案例，用于模型迭代

### 挑战3：快速迭代 vs 长周期研究

**传统产品**：6个月发布一次
**AI产品**：每周更新模型

**应对策略**：
- **持续研究面板**：招募100-500名深度用户，建立长期研究关系
- **快速定性脉冲**：每周5场15分钟mini访谈（替代传统1小时访谈）
- **2026案例**：豆包建立"超级用户群"（3000人微信社群），每周发布新功能→收集反馈→72小时内迭代

## 方法选择决策树

面对具体研究问题，如何选择方法？用这个决策树：

```
研究目标是什么？
    │
    ├─ 我不知道问题在哪 (探索)
    │       ├─ 样本容易获取? → YES → 焦点小组
    │       └─ 样本难获取? → YES → 深度访谈
    │
    ├─ 我有明确假设要验证 (验证)
    │       ├─ 需要因果推断? → YES → A/B测试
    │       ├─ 需要规模数据? → YES → 问卷调查
    │       └─ 有现成数据? → YES → 行为分析
    │
    ├─ 我要理解用户行为 (理解)
    │       ├─ 低频行为? → YES → 日记研究
    │       ├─ 高频行为? → YES → 行为数据分析
    │       └─ 复杂场景? → YES → 民族志
    │
    └─ 我要优化具体体验 (优化)
            ├─ UI/交互问题? → YES → 可用性测试
            ├─ 转化率问题? → YES → A/B测试
            └─ 流失问题? → YES → 流失用户访谈 + 行为分析
```

## 混合方法论实战

真实项目中，单一方法很少有效。看2026年中国AI产品的混合方法论案例：

### 案例：通义千问企业版的"办公助手"功能开发

**背景**：阿里想开发"AI办公助手"，但不知道企业用户真正需要什么。

**研究设计**（4阶段混合）：

```
阶段1：探索 (Week 1-2) → 定性
  └─ 深度访谈 15家企业的知识工作者 (HR/财务/运营)
      ├─ 发现：95%时间在"找文件、整理数据、写重复文档"
      └─ 假设：AI如果能"自动整理文档"会最有价值

阶段2：验证规模 (Week 3) → 定量
  └─ 问卷调查 500家企业 (问卷星)
      ├─ 问题："以下AI功能按价值排序"
      ├─ 结果：68%选"文档自动分类"为Top1需求
      └─ 验证：假设成立，值得投入

阶段3：原型测试 (Week 4-5) → 定性 + 定量
  └─ A组：20家企业试用MVP 2周 (日记研究)
      ├─ 定性发现：用户觉得"分类不准"(40%投诉)
      └─ 定量数据：平均每天只用2次（预期10次）
  └─ 迭代：优化分类算法

阶段4：规模验证 (Week 6-8) → 定量
  └─ A/B测试：10万企业用户
      ├─ A组：旧版（人工分类）
      ├─ B组：新版（AI分类）
      └─ 结果：B组文档处理效率提升35%，付费转化率+12%

最终决策：全量上线 "AI文档助手"
```

**关键启示**：
1. **先定性后定量**：不要一上来就发问卷（你不知道该问什么）
2. **小样本快速迭代**：20家企业试用比2000家企业调研更有价值
3. **定量验证商业价值**：最终用A/B测试证明"值得做"

> **💡 实战洞察：别被"样本量"吓到**
> 很多PM觉得"5个用户访谈太少，没有统计意义"。错！Nielsen研究表明：**5个用户就能发现85%的可用性问题**，15个用户能发现95%。定性研究的价值不在样本量，而在**深度和模式识别**。反而是"100人浅层问卷"容易误导——用户会告诉你"想要更快的马"，而你真正该造的是"汽车"。

- MVP后：500人问卷→验证73%用户每周用>3次
- **教训**：不能跳过定性直接问卷，否则问卷题目都不知道怎么设计

5. 你的AI产品有100万DAU，但月留存只有5%（行业平均15%）。设计一个混合方法研究计划找到原因。

<details>
<summary>💡 参考答案</summary>

**三步混合方法**：

**Step 1: 行为数据分析（定量，Week 1）**
- 用神策/诸葛IO分析流失用户行为路径
- 找到"流失高发节点"（如：80%用户在第3天流失，且都在"首次付费弹窗"后流失）
- **输出**：流失漏斗图 + 高危行为特征

**Step 2: 流失用户访谈（定性,Week 2-3）**
- 招募20个"第3-7天流失用户"深度访谈
- 问："为什么看到付费弹窗就走了？"
- **发现**：用户觉得"还没体验到价值就要收费"（价值感知延迟）

**Step 3: A/B测试验证（定量,Week 4-6）**
- A组（对照）：第3天弹付费窗
- B组（实验）：第7天弹付费窗，且前7天每天推送"成功案例"
- **结果**：B组月留存提升到12%

**Kimi 2026真实案例**：就是这么干的，发现"免费期太短"是流失主因。
</details>

```markdown
# 研究计划：AI写作助手用户增长停滞原因诊断

## 1. 研究目标 (SMART)
- **Specific**：找到导致近3个月DAU增长停滞（+2% vs 目标+15%）的Top3原因
- **Measurable**：输出可量化的改进建议（预期提升XX%指标）

## 2. 混合方法设计

### 阶段1：定量诊断 (Day 1-3)
**方法**：行为数据分析（神策数据）
- **分析维度**：新用户激活率、留存曲线、功能使用率
- **对比**：近3个月 vs 去年同期
- **输出**：增长漏斗诊断报告

### 阶段2：定性深挖 (Day 4-8)
**方法**：深度访谈

- **访谈大纲**：
  1. 为什么开始用我们的产品？
  2. 为什么不再用/继续用？
  3. 竞品替代了我们吗？
- **输出**：访谈洞察主题编码

### 阶段3：规模验证 (Day 9-10)
**方法**：问卷调查（问卷星）
- **样本**：500人（流失用户为主）
- **核心问题**：基于访谈发现的假设验证
```

> **💡 实战洞察：方法只是手段，洞察才是目的**
> 2026年有个常见陷阱：PM觉得"我做了10场访谈、500人问卷、3个A/B测试"就是好研究。错！**研究的价值不在于"做了什么方法"，而在于"产生了什么洞察、驱动了什么决策"**。豆包团队有个原则："每个研究必须回答一个决策问题"——不是为了研究而研究，而是为了**更好的产品决策**。记住：老板要的不是"研究报告"，而是"这个功能该不该做？怎么做？"的明确答案。
