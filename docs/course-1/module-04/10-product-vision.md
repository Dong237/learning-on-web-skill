## 什么是产品愿景

### 愿景的本质

**定义**:产品愿景(Product Vision)是对产品3-5年后理想状态的清晰描述

**不是什么**:
- ❌ 不是KPI:"3年内DAU破1亿"(这是目标,不是愿景)
- ❌ 不是口号:"让世界更美好"(太空洞,无法指导决策)
- ❌ 不是功能列表:"我们要做XX功能+YY功能"(这是Roadmap,不是愿景)

**是什么**:
- ✅ 一幅未来图景:"让每个人都能享受AI带来的生产力提升"
- ✅ 用户视角的改变:"从'写作1小时'到'写作10分钟',AI辅助完成90%工作"
- ✅ 可感知的变化:"办公室里,每个人都在用我们的产品"

### 为什么需要愿景?

**对内:凝聚团队**

```
没有愿景的团队:
工程师:"PM又改需求了,烦死了,反正做完拿工资就行"
设计师:"这个功能没意义,但PM要做,算了做吧"
→ 团队执行力低,离职率高

有愿景的团队:
PM:"这个功能虽然难,但它能帮助100万用户节省时间,值得!"
工程师:"理解,我们是在'让每个人享受AI生产力',加班也愿意"
→ 团队有使命感,凝聚力强
```

**对外:吸引用户和投资人**

```
没有愿景的产品:
用户:"这就是个工具,有替代品我就换"
投资人:"看不到长期价值,不投"

有愿景的产品:
用户:"这个产品的愿景我认同,愿意陪伴成长"
投资人:"愿景清晰,团队有信念,值得下注"
```

### 愿景的时间跨度

```
      时间轴
        ↓
现在 ───┼─── 1年 ───┼─── 3年 ───┼─── 5年 ───┼─── 10年
        │            │           │           │
      目标(Goal)    愿景(Vision)           使命(Mission)
        │            │           │           │
      可量化        可描绘      星辰大海      终极理想
```

**示例**:
- **目标**(1年):DAU破100万,付费用户10万
- **愿景**(3-5年):成为中国最好用的AI写作助手,让3000万知识工作者用上
- **使命**(10年+):让AI赋能每个人,消除信息不对称

## 愿景 vs 使命 vs 价值观

### 三者的区别

| 维度 | 愿景(Vision) | 使命(Mission) | 价值观(Values) |
|------|-------------|--------------|---------------|
| **回答问题** | 我们要去哪里? | 我们做什么? | 我们如何做? |
| **时间跨度** | 3-5年 | 10年+ | 永恒 |
| **是否可变** | 可调整 | 基本不变 | 不变 |
| **描述对象** | 产品状态 | 公司存在意义 | 行为准则 |
| **举例(豆包)** | 3年内成为中国最好用AI助手 | 让AI赋能每个人 | 用户至上/创新/诚信 |

### 实战案例对比

**案例1:特斯拉**

```markdown
使命(Mission):
"加速世界向可持续能源的转变"
→ 解答:我们为什么存在?(改变能源格局)

愿景(Vision):
"让电动车成为主流,取代燃油车"
→ 解答:我们要成为什么?(电动车领导者)

价值观(Values):
"创新、速度、用户导向"
→ 解答:我们如何工作?(快速试错、以用户为中心)
```

**案例2:字节跳动**

```markdown
使命:
"Inspire Creativity, Enrich Life"(激发创造,丰富生活)
→ 为什么存在?帮助人们创造和享受内容

愿景(抖音):
"记录美好生活"
→ 要成为什么?全球最大短视频社区

价值观:
"始终创业、追求极致、务实敢为"
→ 如何工作?像创业公司一样快速迭代
```

### 常见误区

**误区1:愿景=使命**

```
❌ "我们的愿景是让世界更美好"
→ 这是使命(太宏大,10年都做不完)

✅ "我们的愿景是3年内成为亚洲最大的AI教育平台"
→ 这才是愿景(3年可实现,清晰具体)
```

**误区2:愿景太保守**

```
❌ "我们的愿景是成为XX领域的参与者"
→ 太弱,无法激励团队

✅ "我们的愿景是成为XX领域的第一名"
→ 大胆,有野心
```

**误区3:愿景太空洞**

```
❌ "我们要做最好的产品"
→ 什么是"最好"?无法衡量

✅ "我们要做中国用户满意度最高的AI对话产品"
→ 有标准(用户满意度),可衡量
```

## 好愿景的6大特征

### 特征1:Bold(大胆)

**定义**:敢于想象不可能

**反例(保守)**:
```
"3年内DAU从10万做到20万"
→ 太保守,无法激发想象
```

**正例(大胆)**:
```
"3年内让1000万中国学生用上AI学习助手"
→ 10万→1000万,100倍增长,大胆!
```

**如何判断"够不够大胆"?**
- 团队听到后,第一反应是"这可能吗?"→ 够大胆
- 团队听到后,说"这很容易"→ 不够大胆

### 特征2:Inspiring(鼓舞人心)

**定义**:让团队愿意为之奋斗

**反例(无感)**:
```
"成为DAU第三名的AI产品"
→ 数字冰冷,无法打动人
```

**正例(鼓舞)**:
```
"让每个打工人下班早1小时,AI帮你搞定重复工作"
→ 有画面感,团队有共鸣
```

**启发性问题**:
- 如果实现这个愿景,世界会有什么不同?
- 用户的生活会如何改变?
- 团队成员会为此骄傲吗?

### 特征3:Achievable(可实现)

**定义**:大胆但不疯狂,有实现路径

**反例(不可信)**:
```
"1年内超越ChatGPT"(创业公司)
→ 资源差距太大,不可信
```

**正例(可信)**:
```
"3年内在中国法律AI领域超越ChatGPT"
→ 聚焦细分赛道,有可能
```

**平衡大胆与可实现**:
```
        可实现性
            ↑
            │
  舒适区   │   甜蜜区 ⭐
(太保守)   │  (大胆+可信)
────────────┼────────────→ 大胆程度
            │
  危险区   │   幻想区
(还可以)   │  (不可信)
            │
```

### 特征4:Clear(清晰)

**定义**:任何人听完都能理解

**反例(模糊)**:
```
"成为行业领先者"
→ 什么行业?领先什么?
```

**正例(清晰)**:
```
"成为中国企业AI客服市场占有率第一"
→ 明确:地域(中国)+ 赛道(企业AI客服)+ 标准(市占率第一)
```

**清晰度测试**:
- 让公司前台/实习生复述愿景
- 如果TA能讲清楚 → 够清晰
- 如果TA一脸懵 → 太复杂

### 特征5:Measurable(可衡量)

**定义**:能判断是否达成

**反例(无法衡量)**:
```
"成为最好的AI产品"
→ 什么是"最好"?谁说了算?
```

**正例(可衡量)**:
```
"成为中国NPS最高的AI对话产品"
→ NPS可量化,第三方可验证
```

**衡量标准示例**:
- 市场份额(占有率第一)
- 用户规模(DAU破1000万)
- 满意度(NPS>50)
- 行业地位(被xx%的500强企业使用)

### 特征6:Time-bound(有时间限制)

**定义**:明确"何时实现"

**反例(无期限)**:
```
"成为全球最大的AI平台"
→ 何时?不知道
```

**正例(有期限)**:
```
"2028年成为亚洲最大的AI教育平台"
→ 2028年,倒逼执行
```

**时间选择**:
- 太短(1年):无法实现大愿景
- 太长(10年):太遥远,无法激励
- 合适(3-5年):可规划,可努力

## 愿景的4层结构

### 第1层:用户画像(Who)

**问题**:我们服务谁?

```markdown
模糊版:
"所有人"(等于没说)

清晰版:
"中国2200万大学生"(具体人群)
```

**画像要素**:
- 人群规模:"2200万大学生"
- 核心特征:"18-25岁,面临就业压力"
- 痛点:"简历、面试准备耗时,需要AI辅助"

### 第2层:问题定义(What Problem)

**问题**:我们解决什么问题?

```markdown
浅层问题:
"写作慢"

深层问题:
"知识工作者每天花2小时在重复性写作(周报/邮件/总结),
 导致真正需要创造力的工作时间不足"
```

**问题公式**:
```
[用户群体] 在 [场景] 遇到 [痛点],导致 [负面后果]

示例:
大学生(用户群体)
在求职季(场景)
不会写简历(痛点),
导致错失面试机会(后果)
```

### 第3层:解决方案(How)

**问题**:我们如何解决?

```markdown
技术视角:
"用AI大模型生成内容"(工程师能懂,用户不懂)

用户视角:
"输入关键词,AI 10秒生成专业简历,
 比人工写作快30倍,质量不输HR推荐"
```

**解决方案要素**:
- 核心机制:AI生成
- 用户体验:10秒完成
- 价值对比:快30倍+质量好

### 第4层:愿景画面(Vision Image)

**问题**:实现后的世界是什么样的?

```markdown
数字版(冰冷):
"3年内DAU破1000万"

画面版(温暖):
"3年后,每10个大学生中,就有8个在用我们的AI简历助手。
 毕业季不再焦虑,因为AI帮你搞定简历、面试准备。
 你只需专注展示真实的自己。"
```

**画面描述技巧**:
- 用"看得见"的场景(办公室/宿舍/地铁上)
- 用"可感知"的变化(从焦虑到自信)
- 用"第二人称"(你/你们)拉近距离

## 2026中国AI产品愿景案例

### 案例1:豆包(字节跳动)

**愿景**:
> "让AI成为每个人的智能伙伴,
> 3年内服务1亿中国用户,
> 让知识工作者的生产力提升30%"

**解析**:
- ✅ Bold:1亿用户(当前1200万,要增长8倍)
- ✅ Inspiring:"智能伙伴"(有温度)
- ✅ Achievable:字节有资源,3年可行
- ✅ Clear:服务对象(中国用户)、标准(生产力+30%)明确
- ✅ Measurable:用户数(1亿)、生产力(30%)可量化
- ✅ Time-bound:3年

**愿景如何指导Roadmap**:
```
愿景:生产力提升30%
   ↓
拆解:写作效率+50%、信息检索速度+3倍、学习效率+40%
   ↓
功能:AI写作、长文档理解、个性化学习路径
   ↓
Roadmap:Q1上线AI写作优化,Q2上线文档问答,Q3上线学习助手
```

### 案例2:Kimi(月之暗面)

**愿景**:
> "让每个人都能轻松理解长内容,
> 3年内成为全球最强的长文本AI"

**解析**:
- ✅ Bold:全球最强(野心)
- ✅ Inspiring:"轻松理解长内容"(解决真痛点)
- ✅ Achievable:已有技术优势(20万字理解)
- ✅ Clear:聚焦"长文本"赛道
- ✅ Measurable:通过Benchmark(长文本理解准确率)衡量
- ✅ Time-bound:3年

**差异化定位**:
```
豆包:全能型助手(对话+写作+搜索)
Kimi:长文本专家(文档理解+知识提取)
通义千问:企业级平台(API+生态)

→ Kimi的愿景聚焦,避免与巨头正面竞争
```

### 案例3:通义千问(阿里)

**愿景**:
> "成为企业首选的AI基础设施,
> 5年内赋能10万家中国企业,
> 让每个企业都能拥有专属AI"

**解析**:
- ✅ Bold:10万家企业(当前5000家,要增长20倍)
- ✅ Inspiring:"专属AI"(定制化,有价值感)
- ✅ Achievable:阿里有云计算+企业客户基础
- ✅ Clear:To B定位,目标10万企业
- ✅ Measurable:企业客户数可统计
- ✅ Time-bound:5年

**To B vs To C愿景的差异**:
```
To C(豆包):
- 强调用户体验("智能伙伴")
- 强调用户规模(1亿用户)

To B(通义千问):
- 强调商业价值("降本增效")
- 强调企业数量(10万家)
```

## 从愿景到执行

### 愿景分解:从愿景到OKR

**示例:某AI学习助手的愿景分解**

```markdown
【愿景】(3年)
"让1000万中国大学生用上AI学习助手,
 让备考效率提升50%"

↓ 分解为3年目标

【年度目标】(Year 1)
- 用户规模:0 → 100万DAU
- 核心功能:上线AI题目生成、错题分析
- 商业模式:探索付费订阅

【年度目标】(Year 2)
- 用户规模:100万 → 500万DAU
- 核心功能:个性化学习路径、AI陪练
- 商业模式:付费用户50万,ARPU ¥30/月

【年度目标】(Year 3)
- 用户规模:500万 → 1000万DAU
- 核心功能:完整学习闭环,生态建设
- 商业模式:收入破¥2亿/年

↓ Year 1分解为OKR

【Q1 OKR】
Objective:完成MVP,验证产品核心价值
Key Results:
- KR1:上线AI题目生成功能,准确率>90%
- KR2:获取10万种子用户,7日留存>40%
- KR3:用户调研NPS>30

【Q2 OKR】
Objective:优化体验,提升留存
Key Results:
- KR1:7日留存从40%提升到50%
- KR2:DAU突破50万
- KR3:付费转化率>3%

... (依此类推)
```

### 愿景沟通:向团队讲愿景

**场景:Kick-off会议,PM向团队宣讲愿景**

```markdown
【错误做法】:直接念PPT

PM:"我们的愿景是成为中国最大的AI学习平台,
    3年内服务1000万用户,Thank you"

团队:(一脸懵)"哦..."


【正确做法】:讲故事

PM:"大家都考过试吧?我上大学时,
    为了备考CET-6,刷了3000道题,
    结果考了58分,差2分没过。

    为什么?因为我盲目刷题,
    没人告诉我哪些是薄弱环节。

    如果当时有AI学习助手:
    - 分析我的错题,发现我听力弱
    - 生成个性化练习,针对性提升
    - 实时反馈,调整学习路径

    我相信,我能考80+分。

    这就是我们要做的:
    让1000万像我一样的大学生,
    不再因为错误的学习方法浪费时间。

    3年后,当学弟学妹说:
    '学长,多亏了你们的产品,我四级一次过了'

    那一刻,我们会知道:
    这3年的加班、996、无数次推倒重来,
    都值得。

    这就是我们的愿景。
    你们,愿意和我一起实现它吗?"

团队:(鼓掌)"干!"
```

**讲愿景的4步框架**:
1. **痛点共鸣**:讲一个团队能感同身受的故事
2. **解决方案**:我们的产品如何解决这个痛点
3. **未来画面**:3年后的世界是什么样的
4. **召唤行动**:邀请团队一起努力

### 愿景测试:3个关键问题

**测试1:电梯测试(Elevator Pitch)**
```
场景:你在电梯里遇到投资人,只有30秒

你能讲清楚愿景吗?

✅ 通过:"我们让1000万大学生用AI提升学习效率50%"
❌ 失败:"嗯...我们是做AI的...教育...那个..."
```

**测试2:妈妈测试(Mom Test)**
```
场景:向你妈妈(完全不懂技术)解释愿景

她能听懂吗?

✅ 通过:"妈,我们做的产品能帮大学生更高效地学习"
❌ 失败:"我们用大模型训练了一个Transformer架构..."
```

**测试3:离职测试(Resignation Test)**
```
场景:核心工程师要离职,你用愿景挽留

他会留下吗?

✅ 通过:"虽然外面工资高2万,但我相信这个愿景,我留下"
❌ 失败:"愿景不错,但我还是要走"
```

## 💡 实战洞察 (AI PM Practitioner Insights)

> **洞察1:愿景不是"写出来的",而是"活出来的"**
>
> 很多公司把愿景印在墙上,但团队根本不信
>
> **真相**:愿景要渗透到每个决策
>
> **案例**:字节跳动的"追求极致"
> - 不是口号,而是每次Code Review都要问"能不能更快?"
> - 不是标语,而是产品发布前CEO亲自试用,提100条优化建议
>
> **启示**:
> - PM做决策时,问自己"这符合我们的愿景吗?"
> - 团队会议,定期回顾"我们离愿景更近了吗?"
> - 招聘时,问候选人"你认同我们的愿景吗?"

---

> **洞察2:中国PM要学会"双语愿景" - 给老板讲商业,给团队讲情怀**
>
> 老板关心:赚钱、市场份额、估值
> 团队关心:使命感、成就感、改变世界
>
> **实战技巧**:
>
> **对老板**(董事会汇报):
> "我们的愿景是3年内成为中国第一的AI学习平台,
>  占领30%市场份额,年收入¥10亿,估值¥100亿"
>
> **对团队**(All Hands):
> "我们的愿景是让1000万大学生不再因为学习方法错误而浪费青春,
>  让AI成为每个学生的私人教练"
>
> **本质相同,表述不同**
> - 老板版:强调商业价值(数字、收入)
> - 团队版:强调社会价值(用户、改变)

## ✅ 自测题 (Self-Check Questions)

<details>
<summary>Q1: 如何判断一个愿景是"大胆"还是"疯狂"?</summary>

**判断标准**:

**大胆(Good)**:
- 团队听到第一反应:"哇,这很难,但也许可以试试"
- 有清晰的实现路径(虽然挑战大)
- 资源虽紧张,但通过努力+创新可实现

**疯狂(Bad)**:
- 团队听到第一反应:"这不可能,你疯了吗?"
- 完全没有实现路径(纯幻想)
- 资源差距太大,即使10倍努力也做不到

**案例对比**:

场景:某创业公司(10人团队,A轮刚融¥1000万)

❌ 疯狂愿景:
"1年内超越ChatGPT,成为全球第一"
→ OpenAI有数千工程师+数十亿美元,你10人团队不可能

✅ 大胆愿景:
"3年内在中国法律AI领域超越ChatGPT"
→ 聚焦细分赛道,有可能(ChatGPT不懂中国法律)

**平衡点**:
```
可实现概率:
- >70%:太保守(轻松达成,不够激励)
- 30-70%:大胆(有挑战,但可信)
- <30%:疯狂(团队不信,执行不力)
```

</details>

<details>
<summary>Q2: 愿景、使命、价值观,哪个应该先定义?</summary>

**答案**:使命 → 愿景 → 价值观(自上而下)

**原因**:

**Step 1:先定使命(Why)**
- 使命回答"我们为什么存在?"
- 是公司的根基,基本不变
- 示例:特斯拉使命"加速世界向可持续能源转变"

**Step 2:再定愿景(Where)**
- 愿景基于使命,定义"3-5年后要成为什么"
- 可随市场调整
- 示例:特斯拉愿景"让电动车取代燃油车"(3-5年目标)

**Step 3:最后定价值观(How)**
- 价值观基于愿景,定义"如何做才能实现愿景"
- 指导日常行为
- 示例:特斯拉价值观"创新、速度、用户导向"

**实际操作**:

对于创业公司:
1. 创始人先想清楚"我为什么要做这个"(使命)
2. 再想"3年后要成为什么"(愿景)
3. 最后定"团队要有什么文化"(价值观)

对于成熟公司:
- 使命已有(如阿里"让天下没有难做的生意")
- PM定产品线愿景即可(如通义千问的愿景)
- 价值观继承公司文化

</details>

<details>
<summary>Q3: 愿景应该由谁来定?CEO、PM还是团队集体讨论?</summary>

**答案**:CEO/创始人定方向,PM细化,团队共创

**角色分工**:

**CEO/创始人**(公司愿景):
- 定义公司级愿景(如"成为中国最大的AI平台")
- 时间跨度:5-10年
- 自上而下,不可商量(创始人的vision)

**PM**(产品愿景):
- 基于公司愿景,定义产品线愿景(如"AI对话产品3年内服务1亿用户")
- 时间跨度:3-5年
- 可与团队讨论,但PM拍板

**团队**(共创细节):
- 参与讨论愿景的可行性
- 提供执行路径建议
- 最终认同并承诺执行

**流程示例(某AI产品)**:

Week 1:CEO宣布公司愿景
"公司愿景:成为中国最大的AI教育平台"

Week 2:PM起草产品愿景
"我们AI学习助手的愿景:3年内服务1000万大学生"

Week 3:团队Workshop
PM:"大家觉得这个愿景可行吗?"
工程师:"1000万用户,服务器成本¥5亿,预算够吗?"
运营:"1000万用户,我们运营团队要扩到50人,招得到吗?"

PM调整:"好,我们分阶段,Year 1目标100万用户,再规划后续"

Week 4:最终确认
PM正式发布愿景(PPT+All Hands会议)
团队承诺执行

</details>

<details>
<summary>Q4: 如果市场变化,愿景是否应该调整?如何调整?</summary>

**答案**:可以调整,但要慎重(1-2年调整1次,不要频繁)

**何时应该调整愿景?**

**场景1:市场环境巨变**
```
案例:2023年ChatGPT发布

之前愿景(某AI公司):
"成为中国最大的AI聊天机器人平台"

问题:ChatGPT重新定义了AI对话,原愿景过时

新愿景:
"成为中国最好用的AI智能助手(对标ChatGPT)"

调整理由:技术突破改变了市场定义
```

**场景2:战略转型**
```
案例:某AI产品从To C转To B

原愿景:
"让1000万个人用户用上AI"

问题:C端获客成本太高,转向B端

新愿景:
"让10万家企业用上专属AI"

调整理由:商业模式转型
```

**场景3:愿景已实现**
```
案例:抖音的愿景演变

2018愿景:"让1亿中国用户记录美好生活"
2020:达成(DAU 6亿)

2020新愿景:"成为全球最大的短视频社区"
2024:达成(全球DAU 10亿)

2026新愿景:"成为全球最大的兴趣分享平台"
```

**何时不应该调整?**

❌ 遇到短期困难就放弃
```
错误:"Q1目标没达成,愿景太激进,改成保守点的吧"
→ 坚持不够,团队信心受挫
```

❌ 频繁调整(每季度改一次)
```
错误:Q1一个愿景,Q2又改了
→ 团队无所适从,不再相信愿景
```

**调整流程**:

1. **评估必要性**(3个月)
   - 是暂时困难,还是方向错误?
   - 与核心团队深度讨论

2. **征求团队意见**(1个月)
   - 全员Workshop,收集反馈
   - 是否有更好的方向?

3. **正式宣布调整**(1周)
   - 坦诚解释"为什么调整"
   - 展示新愿景的合理性
   - 重新凝聚团队

4. **更新所有材料**(1周)
   - 官网、PPT、招聘JD
   - 对外沟通口径统一

</details>
